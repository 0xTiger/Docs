## .JPF вариант № ##

Файл с расширением .jpf является расширением системы кодирования изображений JPEG 2000 ISO/IEC 15444 и упоминается как Часть 2 ISO/IEC 15444-2. Он определяет и специфицирует набор методов сжатия без потерь (с сохранением битов) и с потерями для кодирования непрерывных тонов, двухуровневых, полутоновых, цветных цифровых неподвижных изображений или многокомпонентных изображений. Первая часть стандарта ISO/IEC 15444-1 относится к [JP2](/ru/image/jp2/), который использует технологию вейвлетов для кодирования контента без потерь и является основой для форматов файлов изображений JPEG 2000. Формат файла JPF не получил теплого приема из-за широкого использования формата JPEG. Файлы JPF можно открывать с помощью популярных приложений для обработки изображений, таких как Adobe Photoshop 2020, Adobe Illustrator 2020 и CorelDraw Graphics Suite 2020.

## Краткая история

В 2000 году комитет Объединенной группы экспертов по фотографии разработал JP2 с целью улучшить свой собственный стандарт JPEG на основе дискретного косинусного преобразования с помощью этого нового метода на основе вейвлета. Комитет JPEG стремился предоставить свои базовые методы бесплатно. В лицензии JP2, выиграв конкуренцию среди 20 компаний, они выиграли с минимальным перевесом. JPEG 2000 был объявлен стандартом ISO, хотя большинство веб-браузеров не готовы принять JPEG 2000 с 2017 года. В 2004 году формат ISO/IEC 15444-2 был публично принят в качестве расширения формата файла JP2.

## Формат файла JPF

Формат файла JPF определяет расширенные процессы декодирования для преобразования сжатых данных изображения для реконструкции. Это расширенный формат файла, который определяет расширенный синтаксис кодового потока, содержащий информацию для интерпретации сжатых данных изображения. Этот расширенный стандарт определяет контейнер для хранения метаданных изображения и предоставляет руководство по расширенным процессам кодирования для преобразования данных исходного изображения в данные сжатого изображения.

### Организация файлов

JPF — это формальный формат файлов хранения, когда файлы JPX хранятся в компьютерных файловых системах. Кроме того, в других Рекомендациях/международных стандартах могут быть определены другие поля для использования в файлах JPX. Однако вся информация, содержащаяся в файле JPX, должна быть в формате коробки; байт-потоки не в формате коробки не должны быть найдены в файле. Двоичная структура блока в файле JPX идентична структуре, определенной в формате файла [JP2](/ru/image/jp2/).

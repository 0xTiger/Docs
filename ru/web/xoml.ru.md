## .XOML вариант №

XOML — это аббревиатура от Extensible Object Markup Language и формат сериализации для объектов рабочего процесса Windows Workflow Foundation. Основанный на **[XAML](/ru/web/xaml/)**, он в основном используется для создания пользовательских интерфейсов в простом **[XML](/ru/web/xml/)**. Они используются для объявления рабочего процесса для пользовательского интерфейса и компилируются вместе с отдельным файлом, содержащим логику реализации. Он содержит древовидную структуру, которая определяет корневой узел рабочего процесса, вложенные подэлементы и встроенные сегменты кода. Когда новый рабочий процесс создается с помощью Visual Studio для .NET, у вас есть возможность выбрать, должен ли он быть в формате кода или в формате с разделением кода. Если вы выберете более позднюю версию, IDE создаст для вас два файла; один в формате XOML (состоит из макета рабочего процесса и настроек), а другой в формате [.CS](/ru/programming/cs/)/[.VB](/ru/programming/vb/), где находится программный код.

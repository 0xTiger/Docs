## .AC вариант №

Файл AC — это файл сценария Autoconf. **Autoconf** — это расширяемый пакет макросов M4, которые создают сценарии оболочки для автоматической настройки пакетов исходного кода программного обеспечения. Эти сценарии могут адаптировать пакеты ко многим типам UNIX-подобных систем без ручного вмешательства пользователя. Autoconf создает сценарий конфигурации для пакета из файла шаблона, в котором перечислены функции операционной системы, которые может использовать пакет, в виде вызовов макросов M4.

Для создания сценариев конфигурации с помощью Autoconf требуется GNU M4. Вы должны установить GNU M4 (по крайней мере версии 1.4.6, хотя рекомендуется 1.4.13 или выше) перед настройкой Autoconf, чтобы сценарий конфигурации Autoconf мог его найти. Сценарии конфигурации, создаваемые Autoconf, автономны, поэтому их пользователям не нужно иметь Autoconf (или GNU M4).

## Как открыть файл AC?

AC означает автоматическую настройку. Это сценарий, созданный GNU Autoconf, который позволяет адаптировать исходный код программного обеспечения к различным Posix-подобным системам путем тестирования необходимых функций в пакете. Сценарий называется файлом AC.

## Основные компоненты Autotools

Понимание автоинструментов GNU (`automake`, `autoconf` и т. д.) может быть полезным при работе с ebuild-файлами:

Autotools — это набор связанных пакетов, которые при совместном использовании устраняют большую часть трудностей, связанных с созданием переносимого программного обеспечения. Эти инструменты вместе с несколькими относительно простыми исходными файлами используются для создания системы сборки пакета.

**Базовый обзор того, как основные компоненты автоинструментов сочетаются друг с другом.**

![A basic overview of how the main autotools components fit together](https://devmanual.gentoo.org/general-concepts/autotools/diagram.png "A basic overview of how the main autotools components fit together")

В простой настройке:

- Программа `autoconf` создает сценарий конфигурации либо из `configure.in`, либо из `configure.ac`.
- Программа automake создает Makefile.in из Makefile.am.
- Сценарий `configure` запускается для создания одного или нескольких файлов Makefile из файлов Makefile.in.
- Программа make использует Makefile для компиляции программы.

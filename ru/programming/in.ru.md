## .IN вариант №

Файл IN — это входной файл, который GNU Autoconf использует при создании приложения. При запуске процесса настройки сценарий Autoconf (файл .AC) может ссылаться на один или несколько файлов «.in» или «.h.in». Файлы IN определяют переменные, на которые ссылаются во время установки программы, такие как номер версии приложения и дата выпуска. Файлы IN также можно использовать для определения наличия или отсутствия функций в конкретной установке. Такие приложения, как Notepad и Notepad++, можно использовать для открытия и редактирования IN-файлов.

## Формат файла IN — дополнительная информация

Autoconf — это компонент системы сборки GNU, который создает пользовательские сборки приложений для различных платформ. Это один из нескольких инструментов сборки GNU. Другие инструменты включают Automake, Gnulib и Libtool. Без необходимости ручного вмешательства пользователя эти сценарии могут адаптировать пакеты к широкому спектру UNIX-подобных систем. Autoconf создает сценарий конфигурации пакета из файла шаблона, в котором перечислены функции операционной системы, которые пакет может использовать в виде вызовов макросов M4.

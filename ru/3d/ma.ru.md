## .MA вариант №

Файл с расширением .ma представляет собой файл 3D-проекта, созданный с помощью приложения Autodesk Maya. Он содержит большой список текстовых команд для указания информации о файле. Файл **.ma** можно открыть и отредактировать в любом текстовом редакторе, чтобы исправить любые проблемы с командами в случае повреждения файла. Эти файлы содержат информацию для определения информации о 3D-сцене, такой как геометрия, освещение, анимация и рендеринг.

## Формат файла МА

Файлы MA сохраняются на диск в текстовом формате ASCII, в отличие от файлов MB, которые сохраняются на диск в двоичном формате. Подробный справочник по формату файла MA доступен в [Документации Autodesk Maya] (https://download.autodesk.com/us/maya/2010help/index.html?url=Glossary_M_ma_file_format.htm,topicNumber=d0e192001) и может быть указан для справки разработчика.

### Заголовок файла MA

Заголовок файла MA начинается с раздела комментариев, который содержит информацию о создании файла и дату изменения. Читатели файлов Maya игнорируют этот блок, так как он используется только в информационных целях. Однако заголовок должен начинаться с первых шести символов как «//Maya».

Заголовок файла ASCII выглядит следующим образом.

```
//Maya ASCII 1.0 scene
//Name: solstice.ma
//Last modified: Sun, Dec 21, 97 10:18:26 AM
```
### Ссылки на файлы

Раздел файловых ссылок файла .ma содержит информацию обо всех других файлах Maya, на которые есть ссылки в этом файле. Каждый указанный файл может быть прочитан с помощью одной файловой команды, содержащейся в том же файле. Синтаксис команды файла при использовании для ссылки:

```
file -r -rpr prefixString fileName;
```
или же

```
file -r -ns nameSpace fileName
```
Вот пример строки.

```
file -r -rpr "solar" "/u/sally/work/solar.ma";
```
Этот пример показывает, что объект солнца, содержащийся в файле `solar`, будет доступен в этом файле по имени "solar_sun".

### Требования

Раздел требований файла .ma состоит из ряда обязательных команд и сообщает информацию о May, такую как информация о версии и плагине, необходимая для чтения файлов. Пример раздела требований выглядит следующим образом.

```
requires maya "2.0";
requires specialPlugIn "1.2";
```


В следующем разделе указаны требования. Он состоит из серии требуемых команд. Этот раздел файла сообщает Maya, какое программное обеспечение необходимо для правильной загрузки файла. Конкретно какая версия Maya и какие плагины.

## .KMZ вариант №

Файл KMZ (KML Zipped) представляет собой заархивированный файл [KML](/ru/gis/kml/), содержащий геопространственную информацию, которую можно просмотреть в таких ГИС-приложениях, как Google Earth. Информация о метках представлена в файле в виде широты и долготы вместе с произвольным именем. Единый упакованный файл KMZ можно легко передать другим пользователям. Файлы KMZ могут также включать данные 3D-модели для географического представления модели. Файл KMZ можно открыть на Картах Google, сохранив файл в Интернете, а затем введя URL-адрес в поле поиска Карт Google.

## Структура файла ##

Содержимое файла MKZ состоит из основного файла KML и нуля или более связанных файлов. Его можно распаковать с помощью стандартной утилиты распаковки, такой как WinZIP. Формат файла KMZ также сжимается в архив с коэффициентом сжатия 10:1. Вы можете экспортировать данные из Google Планета Земля, как приложения, непосредственно в формат файла KMZ. Основной файл KML называется **doc.kml**. При упаковке файла KMZ к нему можно добавить более одного файла KML, но это не принесет пользы, поскольку Google Планета Земля ищет первый файл KML при открытии файла KMZ и считывает его. Он игнорирует любые другие файлы KML, найденные в архиве. Чтобы быть уверенным, что нужный файл KML будет прочитан Google Планета Земля, рекомендуется помещать в файл KMZ только один файл KML.

Изображения, модели, текстуры, звуковые файлы и другие ресурсы, указанные в файле doc.kml, помещаются в другую подпапку внутри основной папки. Это также может потребовать некоторой сложности в зависимости от количества вспомогательных файлов. Ссылки на эти составляющие ресурсы могут быть относительными или абсолютными.

### Относительная ссылка ###

Когда ресурсы размещаются рядом с основным doc.kml внутри подпапки в основной папке, относительная ссылка может указывать на эти вспомогательные файлы, как показано в следующем примере (только для значка).

```
<IconStyle>
  <scale>1.1</scale>
  <Icon>
    <href>files/icon_surfing.png</href>
  </Icon>
</IconStyle>
```

### Абсолютная ссылка ###

На ресурсы также можно ссылаться абсолютно. Абсолютные ссылки содержат полный URL связанного файла. Когда файлы размещаются на центральном сервере, абсолютные ссылки гарантируют, что они останутся однозначными по сравнению с относительными ссылками. Не рекомендуется указывать абсолютные ссылки на локальный файл, поскольку эти ссылки будут разорваны при перемещении файлов в новую систему. Пример абсолютной ссылки выглядит следующим образом:

```
<Icon>
  <href>http://maps.google.com/mapfiles/kml/pushpin/ylw-pushpin.png</href>
</Icon>
```

## .HDR вариант №

Файл HDR — это файл заголовка ГИС, который содержит информацию заголовка для файла полосы с чередованием строк (.BIL). Он описывает данные изображения и имеет то же имя, что и файл изображения. Файл HDR состоит из набора записей, каждая из которых описывает определенный атрибут изображения. Файл HDR описывает макет и форматирование файла BIL для перевода в координаты реального мира в сочетании с отдельным файлом географической привязки.

## Формат HDR-файла

Файлы HDR сохраняются в формате текстового файла ASCII. Он содержит набор записей, каждая из которых описывает определенный атрибут изображения. Каждый файл HDR имеет следующий формат:

```
<keyword> <value>
```

`<keyword> ` - указывает конкретный атрибут, который устанавливается
`<value> ` - Это значение, которому присваивается атрибут.

Ограничений на порядок записей в заголовке нет. Однако каждая запись должна находиться на отдельной строке строки, чтобы соответствовать методу анализа, используемому устройствами чтения HDR.

## Сводка ключевых слов, используемых в файле HDR

|Ключевое слово |Приемлемое значение |По умолчанию|
---|---|---|
|nrows |любое целое число > 0| никто
|ncols |любое целое число > 0| никто
|nbands |любое целое число > 0| 1
|nбиты |1, 4, 8, 16, 32| 8
|байтовый порядок| I = Intel; M = Motorola |то же, что и хост-машина|
|макет| бил, бип, бск| законопроект |
|пропустить байты| любое целое число ≥ 0| 0
|ulxmap |любое вещественное число| 0|
|ulymap |любое вещественное число |nrows - 1|
|xdim |любое вещественное число| 1
|ydim |любое вещественное число| 1
|bandrowbytes |любое целое число > 0| наименьшее целое число ≥ (ncols x nbits)/8|
|totalrowbytes |любое целое число > 0| для bil: nbands x Bandrowbytes; для bip: наименьшее целое число ≥ (ncols x nbands x nbits)/8|
|запрещённая полоса |любое целое число ≥ 0| 0|

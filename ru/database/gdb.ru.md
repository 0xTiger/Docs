## .GDB вариант №

Файловая база геоданных ESRI (FileGDB) представляет собой набор файлов в папке на диске, которые содержат связанные геопространственные данные, такие как наборы данных объектов, классы объектов и связанные таблицы. Для его работы требуется, чтобы некоторые другие файлы хранились вместе с файлом .gdb в том же каталоге. Запросы могут выполняться в файле .gdb для управления как пространственными, так и непространственными данными.

## Формат файла GDB — дополнительная информация

Файловые базы геоданных состоят из семи системных таблиц и пользовательских данных. Пользовательские данные могут храниться в следующих типах наборов данных:

* Класс объектов
* Набор данных функций
* Набор данных мозаики
* Каталог растров
* Набор растровых данных
* Схематический набор данных
* Таблица (непространственная)
* Ящики для инструментов

Наборы пространственных данных могут содержать классы пространственных объектов, а также следующие типы наборов данных:

* Вложения
* Аннотации, связанные с функциями
* Геометрические сети
* Наборы сетевых данных
* Посылка ткани
* Классы отношений
* Ландшафты
* Топологии

По умолчанию максимальный размер наборов данных в файловых базах геоданных составляет 1 ТБ. Максимальный размер может быть увеличен до 256 ТБ для больших наборов данных (обычно растровых). Это контролируется ключевым словом конфигурации. Дополнительные сведения см. в разделе Ключевые слова конфигурации для файловых баз геоданных.

Файловые базы геоданных также могут содержать подтипы и домены и участвовать в репликации открепления/возврата и односторонних репликах.

Доступ к файловой базе геоданных может осуществляться одновременно несколькими пользователями. Если пользователи редактируют, они должны редактировать разные наборы данных.

## Спецификации формата файла GDB ##

Файл GDB является собственным форматом ESRI, и его спецификации не являются общедоступными. По этой причине детали формата файла для FIle GDB не могли быть задокументированы нигде, кроме некоторых источников, которые сделали это [частично](https://github.com/rouault/dump_gdbtable/wiki/FGDB-Spec) путем обратного проектирования. .

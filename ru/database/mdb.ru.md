## .MDB вариант №

Файл с расширением .mdb представляет собой файл базы данных Microsoft Access, который представляет собой систему управления реляционными базами данных (RDBMS). Он хранит данные в таблицах базы данных, которые связаны друг с другом первичными и внешними ключами. Файл MDB содержит полную структуру таблиц базы данных, запросов, хранимых процедур. MDB является форматом файлов по умолчанию для Microsoft Access 2003. В боковых версиях Microsoft Access используются форматы файлов [ACCDB](/ru/database/accdb/), которые являются последним форматом файлов на сегодняшний день. Файлы MDB можно открывать с помощью таких приложений, как Microsoft Access, MDB Viewer, MDBOpener, и их можно конвертировать в форматы ACCDB, [CSV](/ru/spreadsheet/csv/), Excel и т. д.

## Формат файла MDB

Для формата MDB доступны общедоступные спецификации, и он остается собственным файловым форматом Microsoft. Microsoft, однако, предоставляет доступ к файлу MDB с помощью стандарта Open Database Connectivity (ODBC) и Visual Basic для приложений (VBA). Неофициальное руководство по MDB содержит краткое неформальное описание формата MDB, основанное на обратном проектировании, и с ним можно ознакомиться, чтобы узнать о спецификациях.

### страниц

Согласно неофициальному руководству по MDB, файл MDB состоит из страниц фиксированного размера (2048 байт для Jeb DB3 и 4096 байт для Jet DB 4). Первый байт указывает тип страницы. Ниже приведены основные типы страниц:

«Первая страница»: содержит информацию о заголовке базы данных, которая также включает идентификацию версии Jet DB, с которой совместим файл. Кроме того, он также включает информацию о безопасности файлов и карту использования страниц.

`Страница определения таблицы:` Страница определения таблицы указывает столбцы, типы данных, индекс и другую информацию. При необходимости он использует дополнительные страницы и включает карту страниц, содержащих данные строки для этой таблицы.

`Страницы данных:` Страницы данных — это фактические контейнеры данных, в которых данные хранятся в виде строк. Он использует вспомогательные страницы для хранения длинных значений данных.

Одна база данных Microsoft Access может состоять из нескольких файлов, что позволяет превысить ограничения на размер файлов и таблиц. Это облегчает размещение форм и кода во внешнем файле MDB на рабочем столе пользователя и данных в других внутренних файлах MDB на серверах, подключенных к сети.

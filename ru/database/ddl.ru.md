## .DDL вариант №

Файл с расширением .ddl — это файл языка определения данных, который используется для определения схемы базы данных. Он содержит операторы/команды для работы со структурами базы данных, такими как таблицы, столбцы, записи и другие поля. Команды в файле DDL написаны на [SQL](/ru/database/sql/) и могут выполнять такие операции, как создание таблицы в базе данных, удаление и обновление. Схема базы данных принадлежит тому, кто ее создал, и с ней могут выполняться все операции CRUD. Популярными приложениями, которые могут создавать и открывать файлы DDL, являются текстовый редактор Windows, Jetbrains Intellij Idea и EclipseLink.

## DDL-команды

Один файл DDL может содержать несколько команд, которые благодаря правильному синтаксису будут выполняться последовательно и вносить изменения в схему, такие как создание наборов символов и таблиц, удаление таблиц, переименование и изменение таблиц. Следующие команды DDL обычно используются при работе со схемой базы данных.

CREATE — используется для создания базы данных или ее объектов (например, таблицы, индекса, функции, представления, процедуры хранения и триггеров).

`DROP` – используется для удаления объектов из базы данных.

`ALTER` — используется для изменения структуры базы данных.

`TRUNCATE` — используется для удаления всех записей из таблицы, включая удаление всех пробелов, выделенных для записей.

`COMMENT` – Добавляет комментарии в словарь данных.

`RENAME` – переименовывает существующий объект в базе данных.

## Пример DDL

В следующем примере показан оператор DDL для команды CREATE, которая создает таблицу и определяет ее поля.

```
CREATE TABLE employees (
    id            INTEGER       PRIMARY KEY,
    first_name    VARCHAR(50)   not null,
    last_name     VARCHAR(75)   not null,
    fname         VARCHAR(50)   not null,
    dateofbirth   DATE          not null
);
```

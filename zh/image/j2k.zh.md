## 什么是一 .j2k 文件？

**J2K** 文件是使用小波压缩而不是 DCT 压缩压缩的图像。联合图像专家组 (JPEG) 2000 文件使用此文件格式。 J2K 文件以 XML 格式存储有关图像文件的元数据信息，这与使用 EXIF 格式的 .jpeg 或 .jpg 不同。 J2K 文件支持 15 位颜色、Alpha 透明度和无损压缩。有几种商业 API 可用于解码 JPEG 2000 图像，例如 J2K-Codec。可以使用标准图像查看器在 Windows 操作系统上打开 J2K 文件。

## J2K 文件格式##

J2K 文件格式与 JPEG 2000 相同，通常保存为 .jp2 和 .jpc。这使得 J2K 文件遵循以 XML 格式编码元数据的相同方法，其中标准 12234-1 用作 Exif 标记和 XML 组件之间的引用。 JPEG 2000 part-2 扩展进一步改进了它，将动画机制和代码流配置组合到一个图像中。此类扩展文件格式文件保存为 .jpx。

### JPEG2000 文件的布局###

JPEG2000 支持基于可扩展文件格式一致性的各种应用程序。虽然最简单的类型可以包含单个图像，但更复杂的类型可以包含一系列图像，彼此堆叠或基于时间排序。

#### JP2 盒子####
它是 JP2 文件格式的顶级构建块，在标题中包含类型和长度字段，以及数据部分。最值得注意的盒子类型是连续码流盒子。此框在其数据部分存储 JPEG2000 码流。

#### JPEG2000 码流####

JPEG2000 CodeStream 是解码 JPEG2000 压缩图像所需的字节序列。如果该文件除了此码流之外没有其他任何内容，则称为原始码流文件。通常JPEG码流是JPEG2000压缩算法在图像上的应用，虽然它不是唯一的方法。

#### 瓷砖零件####

JPEG2000 编码图像是称为数据包的数据单元的集合。这些数据包保存在称为 tile-parts 的数据包组内的码流中。在对图像进行编码之前，编码器将图像划分为块的矩形网格，称为块，其中每个块被单独编码，而与其他块无关。

## J2K 压缩##
JPEG 2000 使用小波压缩技术使其速度更快，因为在查看器显示图像的任何视口或窗口中显示的像素相对较少。可以通过以下事实来强调这一点：对于非常大尺寸的图像（以千兆字节为单位），屏幕上只会出现几兆字节的像素。这有助于仅快速获取和渲染填充显示像素所需的那部分图像数据。这也需要高速解压缩技术来加速图像获取机制，以便即时创建所需的图像。

J2K 利用快速解压缩的优势，仅获取像素数据的必要信息，以将部分可见图像快速呈现到屏幕上。 J2K 主要用于查看数据而不是编辑数据。

## J2K 识别##
JPEG 2000 文件的签名字节为 6A 50 20 20。

### 哑剧类型###
JPEG 2000 文件的注册 Mime 类型包括：
*图像/JP2
* 图像/jpx
*图像/ jpm
* 视频/mj2

## 对 JPEG 标准的改进##
对 JPEG 标准的改进包括：
* 卓越的压缩性能
* 多分辨率表示
*按像素和分辨率精度进行渐进式传输
* 选择无损或有损压缩
* 错误恢复，灵活的文件格式
* 高动态范围支持
* 侧信道空间信息

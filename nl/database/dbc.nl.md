## Wat is een DBC-bestand?
De bestanden met de extensie .dbc worden ook wel CAN-databasebestanden genoemd. Het DBC-bestand is een eenvoudig tekstbestand dat bestaat uit informatie voor het decoderen van onbewerkte CAN-busgegevens naar fysieke waarden of in voor mensen leesbare vorm. Het DBC-bestand wordt geïntroduceerd, omdat dit de veelgebruikte manier is om de identificatie en vertaling van de gegevens te beheren. Het DBC-bestandstype is ontwikkeld om de middelen voor het bijhouden van gegevens te bieden zoals beschreven in een CAN-netwerk.

## DBC-bestandsindeling
Het DBC-bestandsformaat vertegenwoordigt alleen het lees- of passieve deel, het biedt geen middel om transmissies uit te werken. Om voertuigen te ondersteunen die nog geen specifieke native aanpassing hebben. Dit wordt gedaan met behulp van het algemene DBC-voertuigtype om DBC-bestanden te gebruiken om CAN-gegevens in statistieken te vertalen. Elk bericht in een DBC wordt een C-structuur waarbij de signalen de leden van de C-structuur zijn.

### DBC-structuur
De DBC-gegevens bestaan uit de volgende elementen:

#### Eenvoudig DBC-bericht
Een eenvoudig DBC-bericht bestaat uit het bericht-ID en ten minste één signaal. Hieronder volgt een demonstratie van een bericht dat bestaat uit een enkel 8-bits signaal.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
```
#### Getekende signalen
Een ondertekend signaal kan worden verzonden door een negatieve offset op een signaal toe te passen. Hieronder volgt een voorbeeld van een ondertekend signaal naar het vorige bericht.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_signed : 8|8@1- (1,-128) [0|0] "" DBG
```
#### Fractionele signalen
Een fractioneel signaal kan worden verzonden door het bereik en desgewenst de precisie te bepalen.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_signed : 8|8@1- (1,-128) [0|0] "" DBG
 SG_ IO_DEBUG_test_float1 : 16|8@1+ (0.1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_float2 : 24|12@1+ (0.01,-20.48) [-20.48|20.47] "" DBG
```
#### Opsommingstypen
Een opsommingstype wordt gebruikt wanneer de gebruiker namen wil zien in plaats van getallen. Zie het volgende voorbeeld.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_enum : 8|8@1+ (1,0) [0|0] "" DBG

BA_ "FieldType" SG_ 500 IO_DEBUG_test_enum "IO_DEBUG_test_enum";

VAL_ 500 IO_DEBUG_test_enum 2 "IO_DEBUG_test2_enum_two" 1 "IO_DEBUG_test2_enum_one" ;
```
#### Gemultiplext bericht
het definiëren van gemultiplexte berichten die een enkele bericht-ID gebruiken, is een optie, maar ze worden anders gedecodeerd, afhankelijk van welke gemultiplexte waarde is verzonden. Om hieronder een gemultiplext bericht te verzenden, moet een apart bericht worden verzonden:

```
BO_ 200 SENSOR_SONARS: 8 SENSOR
 SG_ SENSOR_SONARS_mux M : 0|4@1+ (1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_err_count : 4|12@1+ (1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_left m0 : 16|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_middle m0 : 28|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_right m0 : 40|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_rear m0 : 52|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_no_filt_left m1 : 16|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_middle m1 : 28|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_right m1 : 40|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_rear m1 : 52|12@1+ (0.1,0) [0|0] "" DBG
```

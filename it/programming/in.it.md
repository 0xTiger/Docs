## Che cos'è un file IN?

Un file IN è un file di input che GNU Autoconf utilizza durante la creazione di un'applicazione. Quando viene eseguito il processo di configurazione, uno script Autoconf (file .AC) può fare riferimento a uno o più file ".in" o ".h.in". I file IN definiscono le variabili a cui si fa riferimento durante l'installazione del programma, come il numero di versione di un'app e la data di rilascio. I file IN possono essere utilizzati anche per determinare la presenza o l'assenza di funzionalità in una particolare installazione. Applicazioni come Notepad e Notepad++ possono essere utilizzate per aprire e modificare file IN.

## IN Formato file - Ulteriori informazioni

Autoconf è un componente del sistema di build GNU che genera build personalizzate di applicazioni per varie piattaforme. È uno dei numerosi strumenti di compilazione GNU. Altri strumenti includono Automake, Gnulib e Libtool. Senza la necessità dell'intervento manuale dell'utente, questi script possono adattare i pacchetti a un'ampia gamma di sistemi simili a UNIX. Autoconf genera uno script di configurazione del pacchetto da un file modello che elenca le funzionalità del sistema operativo che il pacchetto può utilizzare sotto forma di chiamate macro M4

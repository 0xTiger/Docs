## Czym jest plik DBC?
Pliki z rozszerzeniem .dbc są również znane jako pliki bazy danych CAN. Plik DBC to prosty plik tekstowy, który zawiera informacje do dekodowania nieprzetworzonych danych magistrali CAN do wartości fizycznych lub w formie czytelnej dla człowieka. Wprowadzono plik DBC, ponieważ jest to bardzo powszechny sposób zarządzania identyfikacją i translacją danych. Typ pliku DBC został opracowany w celu zapewnienia środków do przechowywania danych, jak opisano w sieci CAN.

## Format pliku DBC
Format pliku DBC reprezentuje tylko część czytaną lub pasywną, nie zapewnia środków do opracowania transmisji. Aby obsługiwać pojazdy, które nie mają jeszcze określonej natywnej adaptacji. Odbywa się to przy użyciu ogólnego typu pojazdu DBC do wykorzystania plików DBC do tłumaczenia danych CAN na metryki. Każda wiadomość w DBC staje się strukturą C z sygnałami będącymi członkami struktury C.

### Struktura DBC
Dane DBC składają się z następujących elementów:

#### Prosta wiadomość DBC
Prosty komunikat DBC składa się z identyfikatora komunikatu i co najmniej jednego sygnału. Poniżej przedstawiono demonstrację komunikatu składającego się z pojedynczego sygnału 8-bitowego.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
```
#### Podpisane sygnały
Podpisany sygnał może być wysłany poprzez zastosowanie ujemnego przesunięcia do sygnału. Poniżej znajduje się przykład podpisanego sygnału do poprzedniej wiadomości.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_signed : 8|8@1- (1,-128) [0|0] "" DBG
```
#### Sygnały ułamkowe
Sygnał ułamkowy można wysłać, decydując o zasięgu i precyzji, jeśli jest to wymagane.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_signed : 8|8@1- (1,-128) [0|0] "" DBG
 SG_ IO_DEBUG_test_float1 : 16|8@1+ (0.1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_float2 : 24|12@1+ (0.01,-20.48) [-20.48|20.47] "" DBG
```
#### Typy wyliczeniowe
Typ wyliczeniowy jest używany, gdy użytkownik chce zobaczyć nazwy zamiast liczb. Zobacz poniższy przykład.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_enum : 8|8@1+ (1,0) [0|0] "" DBG

BA_ "FieldType" SG_ 500 IO_DEBUG_test_enum "IO_DEBUG_test_enum";

VAL_ 500 IO_DEBUG_test_enum 2 "IO_DEBUG_test2_enum_two" 1 "IO_DEBUG_test2_enum_one" ;
```
#### Wiadomość zmultipleksowana
zdefiniowanie zmultipleksowanych komunikatów wykorzystujących pojedynczy identyfikator komunikatu jest opcją, jednak są one dekodowane w różny sposób w zależności od tego, która zmultipleksowana wartość została wysłana. Aby wysłać wiadomość zmultipleksowaną poniżej, należy wysłać osobną wiadomość:

```
BO_ 200 SENSOR_SONARS: 8 SENSOR
 SG_ SENSOR_SONARS_mux M : 0|4@1+ (1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_err_count : 4|12@1+ (1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_left m0 : 16|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_middle m0 : 28|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_right m0 : 40|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_rear m0 : 52|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_no_filt_left m1 : 16|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_middle m1 : 28|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_right m1 : 40|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_rear m1 : 52|12@1+ (0.1,0) [0|0] "" DBG
```

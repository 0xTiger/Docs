## Was ist eine JHTML-Datei?

Eine Datei mit der Erweiterung .jhtml ist eine [HTML](/de/web/html/)-Datei mit [Java](/de/programming/java/)-Code, der auf dem Server ausgeführt wird, wenn ein Client diese Seite in einem Browser anfordert. Der Server verarbeitet die Anforderungen, führt die in der Funktion enthaltenen Java-Funktionen aus und gibt die Seite an den Browser des Clients zurück. Die in die JHTML-Seiten eingebetteten Java-Objekte werden auf dem Server ausgeführt, um Anforderungen für Seiten dieses Typs zu verarbeiten. JHTML-Dateien können auch über die JDBC-Verbindung (Java [Database](/de/database/) Connectivity) auf Informationen aus der Datenbank zugreifen. JHTML-Dateien können in jedem Texteditor geöffnet und in Webbrowsern wie Google Chrome, Firefox und Safari angezeigt werden.

## JHTML-Dateiformat

JHTML ist eine proprietäre Technologie von ATG und kann mit dem ATG (Art Technology Group) Dynamo Document Editor erstellt werden. JHTML-Dateien werden im Nur-Text-Dateiformat unter Verwendung des Standard-HTML- und Java-Codes geschrieben. Diese enthalten zusätzlich zu proprietären Tags, die auf Java-Objekte verweisen, Standard-HTML-Tags. Wenn eine solche Seite vom Benutzer angefordert wird, leitet der HTTP-Server die Anforderung an einen Java-Anwendungsserver weiter. Die JHTML-Seite wird zuerst in eine .java-Datei konvertiert und kompiliert, um eine [.class](/de/programming/class/)-Datei zu generieren, die als Servlet ausgeführt wird. Dadurch wird ein Strom von Standard-HTTP- und HTML-Daten generiert, der an den anfordernden Browser zur Anzeige für den Benutzer zurückgesendet wird. Dies ist hilfreich, um datenbankbezogene Abfragen auf dem Server auszuführen und das endgültige akkumulierte Ergebnis an den Browser des Clients zurückzugeben.

## Was ist eine IN-Datei?

Eine IN-Datei ist eine Eingabedatei, die GNU Autoconf beim Erstellen einer Anwendung verwendet. Wenn der Konfigurationsvorgang ausgeführt wird, kann ein Autoconf-Skript (.AC-Datei) auf eine oder mehrere ".in"- oder ".h.in"-Dateien verweisen. IN-Dateien definieren Variablen, auf die während der Programminstallation verwiesen wird, z. B. die Versionsnummer und das Veröffentlichungsdatum einer App. IN-Dateien können auch verwendet werden, um das Vorhandensein oder Fehlen von Funktionen in einer bestimmten Installation zu bestimmen. Anwendungen wie Notepad und Notepad++ können zum Öffnen und Bearbeiten von IN-Dateien verwendet werden.

## IN-Dateiformat - Weitere Informationen

Autoconf ist eine Komponente des GNU-Build-Systems, das benutzerdefinierte Builds von Anwendungen für verschiedene Plattformen generiert. Es ist eines von mehreren GNU-Build-Tools. Andere Tools sind Automake, Gnulib und Libtool. Ohne manuellen Benutzereingriff können diese Skripte die Pakete an eine Vielzahl von UNIX-ähnlichen Systemen anpassen. Autoconf generiert aus einer Vorlagendatei ein Paketkonfigurationsskript, das die Betriebssystemfunktionen auflistet, die das Paket in Form von M4-Makroaufrufen verwenden kann

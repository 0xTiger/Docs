## Qu'est-ce qu'un fichier CFG ?

Le fichier CFG dans le contexte du **Valve's Source Engine**, comme on le voit dans des jeux comme Half-Life 2, sert de fichier de configuration. Ces fichiers sont utilisés pour modifier divers paramètres du jeu via une liste de commandes en texte brut. Ces commandes sont généralement organisées à raison d'une par ligne et sont responsables de la personnalisation des aspects du jeu.

Les joueurs et les administrateurs de serveur disposent de plusieurs options pour appliquer les modifications spécifiées dans le fichier `.cfg` :

1. **Exécution manuelle** : les utilisateurs peuvent exécuter manuellement des commandes dans le fichier CFG. Cela signifie qu'ils saisissent chaque commande une par une dans la console développeur du jeu. Cette méthode permet un contrôle précis sur les paramètres qui sont modifiés et à quel moment.
    





2. **Exécution automatique** : Alternativement, les joueurs peuvent opter pour l'exécution automatique. Cela implique de placer le fichier « .cfg » dans le répertoire de jeu approprié. Certains fichiers `.cfg`, comme `autoexec.cfg`, sont exécutés automatiquement au démarrage du jeu. Cela garantit que les paramètres spécifiés sont appliqués sans nécessiter de saisie manuelle à chaque lancement du jeu.

## Le moteur source de valve

Le Valve Source Engine, souvent simplement appelé **Source Engine**, est un moteur de jeu très polyvalent et largement utilisé développé par Valve Corporation. Il a servi de base à de nombreux jeux vidéo populaires et a alimenté de nombreux titres à succès. Voici quelques points clés concernant Valve Source Engine :

1. **Historique** : Le moteur Source a été introduit pour la première fois avec la sortie du jeu de Valve "Counter-Strike 1.6" en 2004. Depuis lors, il a subi plusieurs mises à jour et révisions, la version la plus récente étant connue sous le nom de Source 2.0.
    





2. **Jeux notables** : Le moteur source a été utilisé dans une variété de jeux acclamés par la critique, notamment :
    





- "Half-Life 2" et ses épisodes
- "Portail" et "Portail 2"
- "Team Fortress 2"
- "Left 4 Dead" et "Left 4 Dead 2"
- "Counter-Strike : Source" et "Counter-Strike : Global Offensive"
- "Dota 2"
3. **Caractéristiques** :
    





- **Flexibilité** : Source Engine est connu pour sa flexibilité permettant aux développeurs de créer un large éventail de genres de jeux, des jeux de tir à la première personne aux jeux de réflexion et bien plus encore.
- **Physique** : il comprend un moteur physique robuste qui permet des interactions d'objets et des effets environnementaux réalistes.
- **Support de modding** : Le moteur dispose d'un solide support de modding qui a conduit à la création de nombreux contenus générés par les utilisateurs et de nombreux mods de jeu.
- **Capacités multijoueurs** : Source Engine a été utilisé pour développer des jeux solo et multijoueurs et offre des capacités réseau étendues pour le jeu en ligne.
    





4. **Graphiques** : Au fil des années, Source Engine a reçu des mises à jour graphiques pour suivre l'évolution des capacités matérielles. Il prend en charge des techniques de rendu avancées telles que l'éclairage HDR (High Dynamic Range) et les ombres dynamiques.

## Comment ouvrir le fichier CFG?

Vous avez la possibilité d'ouvrir et de modifier le fichier `.cfg` à l'aide d'éditeurs de texte comme Microsoft Visual Studio Code ou tout autre éditeur de texte de votre choix. Ces éditeurs de texte fournissent une interface conviviale pour afficher et modifier les commandes en texte brut dans le fichier `.cfg`, vous permettant de personnaliser les paramètres selon vos besoins.

Les programmes qui ouvrent ou font référence aux fichiers CFG incluent

-Code Microsoft Visual Studio
- Bloc-notes
- TexteModifier
- N'importe quel éditeur de texte

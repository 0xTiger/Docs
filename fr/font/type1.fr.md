## Que sont les polices de type 1 ?

Les polices de type 1 sont une technologie Adobe obsolète qui était largement utilisée dans les logiciels de publication de bureau et les imprimantes pouvant utiliser PostScript. Bien que les polices de type 1 ne soient pas prises en charge sur de nombreuses plates-formes modernes, les navigateurs Web et les systèmes d'exploitation mobiles, elles sont toujours prises en charge dans certains systèmes d'exploitation. L'insuffisance de prise en charge des informations Unicode dans les polices de type 1 limite également leur capacité à prendre en charge les jeux de caractères de langue étendus.

## Bref historique

La technologie des polices de type 1 a été lancée en 1984 pour être utilisée avec le langage de description de page PostScript d'Adobe. Au milieu des années 90, Adobe a décidé de se concentrer sur l'utilisation de polices OpenType plus flexibles plutôt que Type 1.

### Quand la prise en charge du type 1 prendra-t-elle fin ?
Les utilisateurs ne pourront plus écrire de contenu à l'aide de polices de type 1 à partir de janvier 2023. D'ici là, aucune modification ne sera apportée.
Certains produits comme les applications Document Cloud continueront d'utiliser les polices de type 1 comme ils l'ont toujours fait.


## Limites des polices de type 1

Certaines des limitations connues du type 1 sont indiquées ci-dessous :

- Il ne permet pas d'inclure plus de 256 glyphes dans une seule police. Les polices CID peuvent faire cela, de nombreux périphériques de sortie peuvent ne pas gérer correctement les polices CID.
- Les polices de type 1 ne sont pas multiplateformes. Une conversion complexe est nécessaire lors de la conversion des polices d'une plate-forme vers d'autres.
- Coller les noms de police à la limite de 8 caractères de l'ancien DOS, ce qui rend difficile de déterminer quelle police est stockée dans un fichier.

## Polices de type 1 sur une machine Windows
Les données de police Type 1 se composent de deux fichiers distincts dans Windows :

1. Le fichier avec l'extension ".PFB" se compose de données de contour.
2. Le fichier avec l'extension ".PFM" contient les données métriques.

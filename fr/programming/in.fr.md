## Qu'est-ce qu'un fichier IN ?

Un fichier IN est un fichier d'entrée que GNU Autoconf utilise lors de la construction d'une application. Lorsque le processus de configuration est exécuté, un script Autoconf (fichier .AC) peut faire référence à un ou plusieurs fichiers ".in" ou ".h.in". Les fichiers IN définissent les variables qui sont référencées lors de l'installation du programme, telles que le numéro de version et la date de sortie d'une application. Les fichiers IN peuvent également être utilisés pour déterminer la présence ou l'absence de fonctionnalités dans une installation particulière. Des applications telles que Notepad et Notepad ++ peuvent être utilisées pour ouvrir et modifier des fichiers IN.

## Format de fichier IN - Plus d'informations

Autoconf est un composant du système de construction GNU qui génère des versions personnalisées d'applications pour diverses plates-formes. C'est l'un des nombreux outils de construction GNU. Les autres outils incluent Automake, Gnulib et Libtool. Sans intervention manuelle de l'utilisateur, ces scripts peuvent adapter les packages à une large gamme de systèmes de type UNIX. Autoconf génère un script de configuration de package à partir d'un fichier modèle qui répertorie les fonctionnalités du système d'exploitation que le package peut utiliser sous la forme d'appels de macro M4

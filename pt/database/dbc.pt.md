## O que é um arquivo DBC?
Os arquivos com extensão .dbc também são conhecidos como arquivos de banco de dados CAN. O arquivo DBC é um arquivo de texto simples que consiste em informações para decodificar dados brutos do barramento CAN para valores físicos ou em formato legível por humanos. O arquivo DBC é introduzido, porque esta é a maneira muito comum de gerenciar a identificação e tradução dos dados. O tipo de arquivo DBC foi desenvolvido para fornecer os meios de manutenção de registros descritos em uma rede CAN.

## Formato de arquivo DBC
O formato de arquivo DBC representa apenas a parte de leitura ou passiva, não fornece meios para elaborar transmissões. Para dar suporte a veículos que ainda não possuem uma adaptação nativa específica. Isso é feito usando o tipo de veículo DBC geral para usar arquivos DBC para traduzir dados CAN em métricas. Cada mensagem em um DBC se torna uma estrutura C com os sinais sendo os membros da estrutura C.

### Estrutura DBC
Os dados DBC consistem nos seguintes elementos:

#### Mensagem simples de DBC
Uma mensagem DBC simples consiste no ID da mensagem e pelo menos um sinal. A seguir está uma demonstração de uma mensagem que consiste em um único sinal de 8 bits.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
```
#### Sinais assinados
Um sinal assinado pode ser enviado aplicando um deslocamento negativo a um sinal. A seguir está um exemplo de sinal assinado para a mensagem anterior.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_signed : 8|8@1- (1,-128) [0|0] "" DBG
```
#### Sinais fracionários
Um sinal fracionário pode ser enviado decidindo o intervalo e a precisão, se necessário.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_unsigned : 0|8@1+ (1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_signed : 8|8@1- (1,-128) [0|0] "" DBG
 SG_ IO_DEBUG_test_float1 : 16|8@1+ (0.1,0) [0|0] "" DBG
 SG_ IO_DEBUG_test_float2 : 24|12@1+ (0.01,-20.48) [-20.48|20.47] "" DBG
```
#### Tipos de enumeração
Um tipo de enumeração é usado quando o usuário deseja ver nomes, em vez de números. Veja o exemplo a seguir.

```
BO_ 500 IO_DEBUG: 4 IO
 SG_ IO_DEBUG_test_enum : 8|8@1+ (1,0) [0|0] "" DBG

BA_ "FieldType" SG_ 500 IO_DEBUG_test_enum "IO_DEBUG_test_enum";

VAL_ 500 IO_DEBUG_test_enum 2 "IO_DEBUG_test2_enum_two" 1 "IO_DEBUG_test2_enum_one" ;
```
#### Mensagem multiplexada
definir uma mensagem multiplexada que usa um único ID de mensagem é uma opção, no entanto, elas são decodificadas de forma diferente dependendo de qual valor multipex foi enviado. Para enviar uma mensagem multiplexada abaixo, uma mensagem separada deve ser enviada:

```
BO_ 200 SENSOR_SONARS: 8 SENSOR
 SG_ SENSOR_SONARS_mux M : 0|4@1+ (1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_err_count : 4|12@1+ (1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_left m0 : 16|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_middle m0 : 28|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_right m0 : 40|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_rear m0 : 52|12@1+ (0.1,0) [0|0] "" DRIVER,IO
 SG_ SENSOR_SONARS_no_filt_left m1 : 16|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_middle m1 : 28|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_right m1 : 40|12@1+ (0.1,0) [0|0] "" DBG
 SG_ SENSOR_SONARS_no_filt_rear m1 : 52|12@1+ (0.1,0) [0|0] "" DBG
```

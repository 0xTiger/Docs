## فایل MANIFEST چیست؟

فایل مانیفست فایلی است که حاوی اطلاعاتی در مورد یک برنامه یا بسته نرم افزاری است. فایل معمولاً با پسوند فایل .manifest نامگذاری می شود. فایل مانیفست اطلاعاتی درباره فایل‌های موجود در بسته، شماره نسخه آن‌ها و هرگونه وابستگی بسته به سایر اجزای نرم‌افزار ارائه می‌کند.

فایل های مانیفست معمولاً در پلتفرم ویندوز استفاده می شوند تا اطمینان حاصل شود که برنامه های نرم افزاری به درستی نصب و پیکربندی شده اند. آنها می توانند برای تعیین مواردی مانند نسخه های کتابخانه های مشترک استفاده شوند، فایل های پیکربندی باید گنجانده شوند و کدام کلیدهای رجیستری باید در حین نصب اصلاح شوند استفاده شوند.

علاوه بر ویندوز، فایل های مانیفست ممکن است در زمینه های دیگر نیز استفاده شوند، مانند برنامه های کاربردی وب یا برنامه های Android. فرمت و محتویات خاص یک فایل مانیفست به پلتفرم و برنامه در حال بسته بندی بستگی دارد.

## اطلاعات بیشتر

فایل های مانیفست با فرمت XML هستند. XML یک زبان نشانه گذاری پرکاربرد برای ایجاد اسناد و داده های ساختاریافته است و اغلب در توسعه نرم افزار برای توصیف پیکربندی ها، تنظیمات و سایر ابرداده ها استفاده می شود.

در زمینه برنامه های نرم افزاری، یک فایل XML مانیفست معمولاً حاوی اطلاعاتی در مورد وابستگی های برنامه، اطلاعات نسخه و سایر تنظیمات پیکربندی است. این فایل برای اطمینان از نصب صحیح برنامه و داشتن تمام اجزا و منابع لازم برای اجرای صحیح استفاده می شود.

فایل مانیفست XML ممکن است در بسته برنامه یا به عنوان یک فایل جداگانه که در حین نصب دانلود می شود گنجانده شود. معمولاً با پسوند فایل .manifest نامگذاری می شود و از فرمت خاصی پیروی می کند که توسط پلتفرم یا چارچوبی که برنامه بر روی آن ساخته شده است تعریف شده است.

به عنوان مثال، در Microsoft .NET Framework، یک فایل مانیفست XML برای توصیف وابستگی ها و اطلاعات نسخه برای یک برنامه استفاده می شود و معمولاً به عنوان بخشی از اسمبلی برنامه گنجانده می شود. این فایل توسط Common Language Runtime (CLR) برای تعیین نسخه های صحیح اسمبلی ها برای بارگیری و اطمینان از اجرای صحیح برنامه استفاده می شود.

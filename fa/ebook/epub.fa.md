## فایل EPUB چیست؟

فایل‌های با پسوند .epub یک فرمت فایل کتاب الکترونیکی هستند که فرمت استاندارد انتشار دیجیتالی را برای ناشران و مصرف‌کنندگان ارائه می‌کنند. این قالب تا به حال به قدری رایج شده است که توسط بسیاری از کتابخوان های الکترونیکی و برنامه های نرم افزاری پشتیبانی می شود. به عنوان مثال، در سیستم عامل مک، نرم افزار **Books** از پیش نصب شده، پشتیبانی از باز کردن چنین فایل هایی را فراهم می کند. علاوه بر این، نرم افزارهای سازگار زیادی برای گوشی های هوشمند، تبلت ها و رایانه ها وجود دارد. استانداردهای فایل EPUB توسط [International Digital Publishing Forum](https://idpf.org/epub/30/spec/epub30-publications.html)(IDPF) حفظ می شود. نسخه EPUB 3 همچنین توسط گروه مطالعه صنعت کتاب (BISG)، یک انجمن پیشرو تجارت کتاب برای بهترین شیوه‌ها، تحقیقات، اطلاعات و رویدادهای استاندارد شده برای بسته‌بندی محتوا، تأیید شده است.

## تاریخچه مختصر فرمت فایل EPUB

* **اکتبر 2007** - EPUB 2.0 تایید شد

* **سپتامبر 2010** - به روز رسانی تعمیر و نگهداری منتشر شد

* **اکتبر 2011** - مشخصات EPUB 3.0 اعمال شد

* **ژوئن 2014** - به روز رسانی تعمیر و نگهداری جزئی برای جایگزینی نسخه 3.0 منتشر شد

* **ژانويه 2017** - EPUB 3.1 فعال شد


## فرمت فایل EPUB

فرمت فایل EPUB آرشیوی است که می توان آن را به پسوند [ZIP](/compression/zip/) تغییر نام داد و با استخراج آرشیو با هر استخراج کننده بایگانی، محتویات آن قابل مشاهده است. این یک فرمت باز مبتنی بر XML است و از فایل های HTML، تصاویر، شیوه نامه های CSS و عناصر دیگر تشکیل شده است. همچنین می توان آن را از طریق تعدادی نرم افزار و API به PDF، Mobi و چندین فرمت فایل دیگر تبدیل کرد.

فرمت فایل EPUB بر اساس سه استاندارد باز است.

### ساختار عمومی باز (OPS) ###

یک فایل EPUB 2.0 از XHTML 1.1 برای ساخت محتوای یک نشریه استفاده می کند. در اصل این بدان معنی است که یک فایل EPUB از یک یا چند صفحه وب تشکیل شده است. حتی اگر می‌توانید کل محتوای یک کتاب یا روزنامه را در یک صفحه قرار دهید، بهتر است که چنین فایلی هم به دلیل عملکرد و هم به دلایل سازگاری از 300K تجاوز نکند. درست مانند صفحات وب معمولی، استایل و چیدمان با استفاده از شیوه نامه های آبشاری (CSS) تعریف می شود. در فایل های EPUB باید از یک زیر مجموعه (سری محدود از دستورات) CSS2 استفاده شود. بسیاری از ویژگی های جدید CSS3، مانند جعبه های گرد یا سایه های باز، هنوز در دسترس نیستند. برای سازگاری به عقب، یک سازنده همچنین می تواند از DTBook به جای XHTML برای رمزگذاری محتوا استفاده کند.

### قالب بسته بندی باز (OPF) ###

این قسمت از مشخصات با اطلاعات ساختاری مانند ابرداده (نویسنده و ناشر کیست، عنوان چیست،...)، مانیفست (لیستی از تمام فایل های داخل یک فایل epub) و فهرست مطالب سروکار دارد. این داده ها همه با استفاده از XML جاسازی شده اند.

### فرمت ظرف باز (OCF) ###

As the above descriptions should have made it clear an EPUB document consists of a series of files. The OCF specs define how all those files end up being packaged in one single container file. ZIP compression is used for this.

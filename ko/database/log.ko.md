## .LOG 파일이란?
확장자가 .log인 파일에는 타임스탬프가 있는 일반 텍스트 목록이 포함되어 있습니다. 일반적으로 특정 활동 세부 정보는 개발자 또는 사용자가 특정 기간에 발생한 일을 추적하는 데 도움이 되도록 소프트웨어 또는 운영 체제에 의해 기록됩니다. 사용자는 텍스트 편집기를 사용하여 이러한 파일을 매우 쉽게 편집할 수 있습니다. 일반적으로 오류 보고서 또는 로그인 활동은 운영 체제에 의해 기록되지만 다른 소프트웨어나 웹 서버도 방문자를 추적하고 대역폭 사용을 모니터링하기 위해 로그 파일을 생성합니다.

## 로그 파일 형식
LOG 파일 형식은 운영 체제에서 발생하는 일반적인 이벤트, 통신 소프트웨어의 다른 사용자 간의 메시지 또는 다른 소프트웨어 실행을 기록합니다. 단순히 특정 타임스탬프의 특정 메시지가 LOG 파일에 기록된다고 말할 수 있습니다. 로깅에 일반적으로 사용되는 용어는 다음과 같습니다.
### 이벤트 로그
시스템의 활동을 추적 및 이해하고 문제를 진단하기 위해 시스템 실행에서 발생하는 이벤트를 기록합니다. 일반적으로 사용자 상호 작용이 거의 없는 응용 프로그램의 경우 복잡한 시스템의 활동을 식별하는 데 중요합니다.
### 트랜잭션 로그
거의 모든 데이터베이스 시스템은 일반적으로 나중에 분석을 위한 감사 추적으로 의도되지 않고 사람이 읽을 수 없는 트랜잭션 로그를 유지 관리합니다. 이러한 로그는 데이터베이스가 충돌 또는 기타 데이터 오류로부터 복구할 수 있도록 하고 데이터를 일관된 상태로 유지하기 위해 기존 데이터에 대한 변경 기록만 유지합니다. 따라서 데이터베이스 시스템에는 일반적으로 일반 이벤트 로그와 트랜잭션 로그가 모두 있습니다.
### 메시지 로그
IRC(Internet Relay Chat), IM(인스턴트 메시징) 프로그램, 채팅 기능이 있는 P2P 파일 공유 클라이언트, 멀티플레이어 게임(특히 MMORPG)에 의해 저장되는 텍스트 통신을 메시지 로그라고 합니다. 이러한 로그는 일반적으로 일반 텍스트 파일로 저장되지만 IM 및 VoIP 클라이언트(예: Skype)는 쉽게 읽거나 암호화를 활성화하기 위해 HTML 파일로 저장할 수 있습니다.
### 서버 로그
서버 로그는 실제로 서버 자체에서 자동으로 수행 및 생성 또는 유지 관리되는 활동 목록을 포함하는 로그 파일입니다. 일반적으로 이러한 파일은 일반 인터넷 사용자가 액세스할 수 없으며 웹마스터 또는 기타 인터넷 서비스 관리자만 액세스할 수 있습니다.

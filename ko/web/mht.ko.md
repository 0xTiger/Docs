## .MHT 파일이란?

확장자가 .mht인 파일은 다양한 유형의 데이터를 단일 파일에 포함하는 MIME 지원 보관 파일 형식입니다. [CSS](/ko/web/css/) 파일 형식의 텍스트, 이미지, 페이지 스타일, JavaScript 및 기타 리소스와 같은 데이터를 임베디드 리소스로 저장할 수 있습니다. MIME 유형이 message/rfc822인 MHT 파일은 HTML 파일의 모든 내용을 저장 장치에 보관할 때 저장하기 위한 단일 보관 파일로 캡슐화합니다. Microsoft Word와 같은 소프트웨어 응용 프로그램을 사용하면 MHT 파일로 내보내 WORD 문서를 MHT로 변환할 수 있습니다. MHT 파일은 Microsoft Internet Explorer 및 Google Chrome과 같은 널리 사용되는 브라우저를 사용하여 열 수 있습니다.

## MHT 파일 형식

[MHTML](/ko/web/mhtml/)과 마찬가지로 MHT도 집계 HTML 문서의 MIME 캡슐화입니다. 인라인 그림, 스타일 시트, 애플릿 등과 같은 MHT 문서 내부의 문서 콘텐츠는 URI를 통해 루트 리소스에 연결되는 MIME 인코딩됩니다. Microsoft Outlook과 같은 이메일 클라이언트는 이메일(일반적으로 [EML](/ko/email/eml/))을 MHT 파일 형식으로 저장합니다. MHT 파일 형식의 전체 사양은 [RFC 822](https://tools.ietf.org/html/rfc822)에 나와 있으며 MHT 파일을 처리할 수 있는 소프트웨어 응용 프로그램 개발에 참고할 수 있습니다.

## MHT와 MHTML의 차이점

온라인 페이지를 저장하는 동안 사용자는 온라인 페이지가 기본 시스템에 저장되어야 하는 파일 형식의 종류를 결정해야 합니다. 가장 일반적으로 사용자는 마크업 언어와 MHTML 파일 형식을 혼동합니다. 그들은 파일 형식이 이들 사이에서 더 건강하다는 것을 보는 것이 번거롭다는 것을 알아차립니다.

사용자가 마크업 언어로 온라인 페이지를 낭비하지 않기로 결정하면 페이지의 다양한 콘텐츠, 즉 텍스트, 그래픽, 애플릿 등에 대해 생성된 완전히 다른 파일 영역 단위에 대한 여러 파일을 포함하는 폴더가 형성됩니다. 반면에 저장 MHTML로 온라인 페이지는 완전한 온라인 페이지를 위한 하나의 파일을 생성합니다. 페이지의 모든 요소와 그래픽, 링크 및 대체 파일 영역 단위가 하나의 파일에 포함되어 있습니다.

당연히 MHT 또는 MHTML 파일을 처리하는 것은 간단합니다. 파일은 전자 메일을 통해 보호되고 공유될 수 있기 때문입니다. 그러나 마크업 언어 파일 영역 단위는 파일 하나라도 손실 또는 삭제되어 사용자에게 쓸모없는 완전한 마크업 언어 폴더를 생성할 수 있으므로 정보 손실 가능성이 있습니다.

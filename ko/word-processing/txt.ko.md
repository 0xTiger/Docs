## .TXT 파일이란?

.TXT 확장자를 가진 파일은 라인 형태의 일반 텍스트를 포함하는 텍스트 문서를 나타냅니다. 텍스트 문서의 단락은 캐리지 리턴으로 인식되며 파일 내용을 더 잘 정렬하는 데 사용됩니다. 표준 텍스트 문서는 다른 운영 체제의 모든 텍스트 편집기 또는 워드 프로세싱 응용 프로그램에서 열 수 있습니다. 이러한 파일에 포함된 모든 텍스트는 사람이 읽을 수 있는 형식이며 일련의 문자로 표시됩니다.

텍스트 파일은 내용의 크기에 제한이 없기 때문에 많은 양의 데이터를 저장할 수 있습니다. 그러나 이러한 대용량 파일을 여는 텍스트 편집기는 이러한 파일을 로드하고 표시하는 데 스마트해야 합니다. 거의 모든 운영 체제에는 텍스트 파일을 만들고 편집할 수 있는 텍스트 편집기가 함께 제공됩니다. 예를 들어 Windows OS에는 이러한 용도로 메모장과 워드패드가 함께 제공됩니다. 마찬가지로 MacOS에는 텍스트 문서를 만들고 편집할 수 있는 TextEdit가 함께 제공됩니다. 그러나 기능 면에서 훨씬 더 발전된 메모장++과 같은 텍스트 문서로 작업할 수 있는 기능을 제공하는 인터넷을 통해 사용할 수 있는 다른 무료 텍스트 편집기도 있습니다.

## 파일 형식 사양 ##

텍스트 파일 형식에는 특별한 파일 형식 사양이 없습니다. 텍스트 파일에는 "텍스트/일반" MIME 유형이 있으며 서식이 거의 또는 전혀 없습니다. 이를 통해 텍스트 편집기는 다른 요구 사항 없이 이러한 파일을 열 수 있습니다. 텍스트 파일의 기본 문자 집합은 텍스트 파일 내용을 만들고 표시하는 데 사용되는 ASCII입니다. 문자는 ASCII 문자 집합을 사용하여 인코딩되지만 ASCII 문자 집합을 사용하여 표현할 수 없는 파운드 기호, 달러 및 유로 기호와 같은 문자에 대한 사용 제한이 있습니다. 따라서 텍스트 파일은 UTF-8이 주로 사용되는 유니코드 형식으로도 저장할 수 있습니다.

### Windows 텍스트 파일 형식 ###

Windows OS의 텍스트 파일은 여러 줄로 구성되며 각 줄은 일련의 문자로 구성됩니다. 각 사용자 암시 줄은 캐리지 리턴(CR)과 줄 바꿈(LF)의 두 문자 조합으로 정의됩니다. Windows 텍스트 파일은 ANSI, OEM, 유니코드 또는 UTF-8 인코딩일 수 있습니다. UTF-16 인코딩은 표현을 위해 2바이트가 필요한 텍스트 파일에 정보를 저장하는 데 도움이 됩니다. 이러한 파일은 일반적으로 파일 내용의 엔디안을 나타내는 BOM(바이트 순서 표시)으로 시작합니다. Windows OS의 다른 응용 프로그램은 정보를 텍스트 파일 형식으로 저장할 수 있지만 응용 프로그램별 텍스트를 나타내기 위해 다른 파일 확장명을 사용합니다. 예를 들어, 프로그래밍 언어는 일반적으로 코드를 텍스트 파일에 저장하지만 고유한 확장자를 사용합니다.

### 유닉스 텍스트 파일 형식 ###

이러한 모든 시스템은 문자가 0개 이상의 행으로 구성된 파일로 텍스트 파일을 세분화합니다. 각 줄은 0개 이상의 줄 바꿈이 아닌 문자와 종료 줄 바꿈 문자(일반적으로 LF)의 시퀀스입니다.

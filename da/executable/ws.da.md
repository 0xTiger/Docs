## Hvad er en WS fil?

En **.WS-fil** på Windows indeholder typisk et eksekverbart script, der er designet til at blive eksekveret gennem **Windows Script Host**. Dette script kan omfatte forskellige elementer og rutiner såsom JScript- og VBScript-rutiner samt XML-elementer. Når du dobbeltklikker på en .ws-fil, vil Windows Script-værten starte udførelsen af scriptet, der er indlejret i filen.

## Om Windows Host Script

Windows Script Host (WSH) er scriptvært udviklet af Microsoft, som giver brugerne mulighed for at køre scripts skrevet på scriptsprog som VBScript (Visual Basic Scripting Edition) og JScript (Microsofts version af JavaScript); det giver rammer for scripting i Windows-operativsystemet, hvilket muliggør automatisering af forskellige opgaver og systemadministration.

Her er nogle vigtige punkter om Windows Script Host:

1.  **Scriptsprog:** WSH understøtter flere scriptsprog, hvor VBScript og JScript er de mest almindeligt anvendte; scripts kan skrives på et af disse sprog for at automatisere opgaver, interagere med operativsystemet eller manipulere data.
    
2.  **Scriptudførelse:** WSH-scripts gemmes typisk i filer med filtypenavne som .vbs for VBScript eller .js for JScript; når du kører et script, fortolker og udfører WSH koden.
    
3.  **Automation:** WSH bruges ofte til automatiseringsformål såsom automatisering af gentagne opgaver, styring af systemindstillinger og udførelse af administrative funktioner; er især nyttig for systemadministratorer og superbrugere.
    
4.  **Konsol- og GUI-scripts:** WSH-scripts kan enten være konsolbaserede eller GUI-baserede; konsolscripts kører i et kommandolinjevindue, mens GUI-scripts kan skabe grafiske brugergrænseflader til mere interaktive applikationer.
    
5.  **Security:** WSH has security features to help prevent malicious scripts from causing harm; users are typically prompted before executing scripts and scripts are subject to certain restrictions to prevent unauthorized access and actions.
    
6.  **Windows Script Host Object Model:** WSH leverer en objektmodel, der tillader scripts at interagere med Windows-operativsystemet; dette inkluderer adgang til filsystemer, indstillinger i registreringsdatabasen, netværksressourcer og andre systemkomponenter.
    
7.  **Scriptudførelse fra kommandolinje:** Du kan udføre WSH-scripts fra kommandolinjen ved at bruge `cscript.exe` eller `wscript.exe` eksekverbare filer, afhængigt af om du vil køre scriptet i et konsol- eller GUI-miljø .

## Hvordan åbner jeg filen WS?

WS filer er almindelige tekstfiler, så hvis du vil åbne eller redigere dem, kan du bruge en hvilken som helst teksteditor, f.eks.

- **Notesblok**
- **Notesblok++**
- **Apple TextEdit**
- **Visuel studiekode**

Bemærk venligst, at hvis du vil køre scriptet inde i WS-filen, kan du blot dobbeltklikke på det eller bruge 'wscript'-kommandoen på kommandolinjen.

## .CFG オプション番号##

拡張子が .cfg のファイルは、「設定」ファイルの一種です。これは一般的に使用されるファイル タイプであり、コンピューター プログラムの構成と設定に関する情報を保存するために使用されます。ほとんどの種類の CFG ファイルはテキスト形式で保存されているため、手動で開くのではなく、テキスト エディターを使用して開く必要があります。ただし、CFG ファイルにはさまざまな種類があり、情報を格納する形式が異なります。 CFG ファイルが提供する機能は、アプリケーションによって異なります。一部のコンピュータ アプリケーションでは、ユーザーはグラフィカルな干渉を使用して構成ファイルの構文を変更または開発できますが、テキスト エディタを使用した変更のみを許可するものもあります。これらのファイルを変更した後、ユーザーはアプリケーションにこれらのファイルを再度読み取り、システムに変更を適用するように指示できます。


## CFG ファイル形式 ##

CFG ファイルは、Unix および Unix ライクなオペレーティング システム、MS-DOS、macOS、Microsoft Windows、IBM OS/2 などのさまざまなオペレーティング システムでサポートされています。これらのファイルが保存され、これらのオペレーティング システムごとに使用される形式は異なります。ほとんどのシステムでは、これらのファイルを人間が判読可能で編集可能なプレーン テキスト形式で使用および保存しますが、ファイルの使用法やオペレーティング システムの要件に応じて、より複雑な形式で保存するシステムもあります。

Unix および Unix ライクなオペレーティング システムでは、ほとんどの CFG ファイルで CFG ファイルのいくつかの異なる形式スタイルが使用されますが、最も一般的な形式は読みやすいプレーン テキスト形式であり、ほとんどすべての形式でコメントを作成および編集できます。これらのオペレーティング システムでの CFG ファイルの最も一般的なファイル拡張子は、CNF、CONF、CF、および INI です。

MS-DOS オペレーティング システムでは、最初は 1 つの構成ファイル形式、つまりプレーン テキストしかありませんでしたが、MS-DOS 6 では、INI 構成ファイル形式が導入されました。

macOS は、標準のプロパティ リスト形式のスタイル構成ファイルを使用します。

Microsoft Windows では、プレーン テキストの INI スタイルの構成ファイルが情報の保存と編集の重要な情報源でしたが、1993 年に新しいデータベース システムが導入され、1993 年以降 Microsoft Windows で構成ファイルの使用が減少しました。


## CFG の例 ##

サンプルの CFG ファイルを以下に示します。

```
#########################
## Settings
##

genome_dir = ~/genome/hg18/

> reads_list1
fastq_100k_1_1.txt
fastq_100k_3_1.txt
<

> reads_list2
fastq_100k_1_2.txt
fastq_100k_3_2.txt
<

read_format = FASTQ
quality_format = phred-33
mapper = bowtie
annotations = all.gene.refFlat.txt
out_path = output
max_intron = 400000
max_multi_hit = 10

```

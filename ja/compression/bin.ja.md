## .BIN ファイルとは何ですか?

Macintosh コンピュータのコンテキストでは、BIN ファイルは MacBinary 形式で保存されたファイルを参照できます。 MacBinary は、インターネット、電子メール、FTP、またはポータブル メディアを介して Macintosh ファイルを転送するために特別に設計されたファイル形式です。 MacBinary の目的は、データ フォークとリソース フォークの両方を含む Macintosh ファイルの完全なファイル構造と属性を 1 つのファイル内に保存することです。

MacBinary 形式は、Macintosh Hierarchical File System (HFS) リソース フォークとデータ フォークを圧縮ファイルにカプセル化することでこれを実現します。ファイルに関する重要なメタデータを含むファインダー ヘッダーも含まれています。 MacBinary は、これらすべてのコンポーネントを 1 つのファイルに結合することで、ファイルの整合性を維持し、異なるプラットフォーム間で簡単に転送および共有できるようにします。

Apple のファイル システムとファイル転送プロトコルの進歩により、BIN ファイルと MacBinary 形式の使用はあまり一般的ではなくなりました。 ZIP などのファイル形式の導入と、HFS+ や APFS などのより新しいファイル システムへの移行により、MacBinary でエンコードされたファイルの必要性が減りました。これらの新しいファイル システムは、ファイル属性、リソース フォーク、データ フォークを処理するより効率的な方法を提供するため、今日のコンピューティング環境では MacBinary 形式の必要性が低くなります。

## BIN ファイル形式 - 詳細情報

Macintosh コンピュータの初期の頃、データ制限のため、ファイルは 2 つの別々の「フォーク」に保存されていました。 「リソース フォーク」には構造化データが含まれ、「データ フォーク」には非構造化データが含まれていました。 Classic Mac OS はこれらのフォークを単一のファイルとして扱いましたが、Mac 以外のシステムにファイルを転送すると、フォークが単一のエンティティとして認識されなかったため、データが失われます。これに対処するために、MacBinary 形式は、Dennis Brothers、Harry Chesley、Yves Lempereur などの個人によって作成されました。 MacBinary は、Mac 以外のシステムに転送できるように、フォークを圧縮して、BIN ファイルと呼ばれる 1 つのファイルに結合しました。 Mac OS に戻ると、フォークは再び分離されます。 2000 年代のフォークベースの HFS からの移行に伴い、MacBinary 形式の使用は大幅に減少しました。現在、Mac 以外のシステムで古いファイルを見つけたり、インターネットからダウンロードしたりしない限り、MacBinary でエンコードされた BIN ファイルに遭遇することはほとんどありません。

MacBinary 形式は、Macintosh システムとファイル処理の変更に対応するために、時間の経過とともにいくつかのバージョンを経てきました。 MacBinary 形式のさまざまなバージョンの一部を次に示します。

- **MacBinary I:** 1980 年代後半に導入された MacBinary の初期バージョン。リソース フォークとデータ フォークを 1 つのバイナリ ファイルに結合し、Macintosh ファイルのクロスプラットフォーム転送を可能にしました。

- **MacBinary II:** 1990 年代初頭にリリースされたこのバージョンは、ファイル タイプや作成者コードなどの追加情報をバイナリ ファイルのヘッダーに追加することにより、元の MacBinary 形式を改良しました。これらのコードは、転送中の Macintosh ファイルの整合性と識別を維持するのに役立ちました。

- **MacBinary III:** 1990 年代半ばに導入された MacBinary III 形式は、バイナリ ファイルのヘッダーにファイル名、ファイルの作成日と変更日などの追加のメタデータを含めることで、以前のバージョンをさらに強化しました。これにより、転送中に Macintosh ファイル属性をより包括的に保存できるようになりました。

- **MacBinary IV:** MacBinary IV は、2000 年代初頭に新たに登場した Mac OS X オペレーティング システムとの互換性の問題に対処するために開発されました。これには、Mac OS X で導入された新しいファイル システム構造と属性に適応するための変更が組み込まれています。

## BIN ファイルを開くには?

MacBinary でエンコードされた BIN ファイルは、さまざまな圧縮ユーティリティを使用して開くことができます。 macOS ユーザーの場合、**Apple Archive Utility** が適切なオプションです。 Windows ユーザーは、**Smith Micro StuffIt Deluxe** などのソフトウェアを利用して、MacBinary でエンコードされた BIN ファイルの内容を抽出できます。 macOS ユーザー向けのもう 1 つのオプションは **The Unarchiver** です。これは、MacBinary を含む複数のファイル形式を処理できる多用途ツールです。

.bin ファイル拡張子は、MacBinary だけでなく、さまざまなファイル形式で使用されることに注意することが重要です。したがって、前述のユーティリティで開けない BIN ファイルが発生した場合は、別の形式で保存されている可能性があります。このような場合、特定のファイル形式を特定し、その形式用に設計された適切なソフトウェアまたはユーティリティを使用してファイルの内容にアクセスする必要がある場合があります。

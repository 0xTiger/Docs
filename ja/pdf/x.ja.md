# PDF/X ファイルとは? #

PDF/X は、2001 年に発行された ISO 15930 標準であり、PDF 機能のサブセットを備えています。この規格は、印刷および出版業界の特定の要件に基づいて確立および公開されました。この規格の要件はすべて、印刷および出版業界の多様なニーズに従って考案されました。 PDF/X では、準拠ファイルが完全である必要があります。つまり、自己完結型です。これには、ページで使用されるフォントなどの要素がドキュメントの一部である必要があります。 3D やビデオなどのコンテンツを PDF/X ドキュメントの一部にすることはできません。 PDF/X ドキュメントに含まれる情報は正確である必要があります。

## PDF/X の標準と改訂 ##

標準の PDF/X ファミリは、それぞれが特殊な結果のために設計されたいくつかのバージョンで構成されています。これらの規格の開発は、印刷および出版業界のさまざまなニーズに対応することを目的としていました。

* `PDF/X-1a` - PDF/X ファミリの最初の標準としても知られる PDF/X-1a では、印刷できるようにするために、すべてのコンテンツが PDF ドキュメントの一部である必要があります。フォント、フォーム、パスワード保護、目に見える注釈などの文書要素は許可されていません。 PDF/X-1a には、透過性やレイヤーの許可など、独自の要件があります。また、印刷要素は CMYK、グレースケール、またはスポット カラーのみを使用する必要があるため、RGB またはデバイスに依存しない色空間は使用されません。すべてのファイルを CMYK (および/またはスポット カラー) で配信し、RGB またはカラー管理データを使用しない交換用です。 PDF/X-1a は、必要な情報が完全であるため、完全な交換とも呼ばれます。
* `PDF/X-3` - 2002 年に導入され、PDF/X-1a の多くの制限が解除されました。これにより、PDF/X-3 のグラフィックスで CMYK、grescale、RGB、Lab、および ICC ベースの色空間を使用できるようになりました。実際には [ICC プロファイル](https://en.wikipedia.org/wiki/ICC_Profile) を使用した PDF 標準 1.3 に基づいています。 PDF ドキュメントに含まれる色に関連する柔軟性とルールが導入されるため、カラー管理とも呼ばれます。
* `PDF/X-4` - 透明度をサポートしているため、PDF-X/4 には出力に必要なすべてのデータが含まれており、平坦化する必要はありません。
* `PDF/X-5` - PDF/X-4 に基づいており、参照 XObject を介した外部グラフィックスのサポートと、レンダリング インテント用の外部 n-colorant プロファイルを追加しています。 PDF バージョン 1.6 を使用した印刷データの部分的な交換に使用します。

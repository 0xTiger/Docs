## Vad är WS fil?

En **.WS-fil** på Windows innehåller vanligtvis ett körbart skript som är utformat för att köras via **Windows Script Host**. Detta skript kan omfatta olika element och rutiner som JScript- och VBScript-rutiner samt XML-element. När du dubbelklickar på en .ws-fil kommer Windows Script Host att initiera körningen av skriptet som är inbäddat i filen.

## Om Windows Host Script

Windows Script Host (WSH) är skriptvärd utvecklad av Microsoft som tillåter användare att köra skript skrivna på skriptspråk som VBScript (Visual Basic Scripting Edition) och JScript (Microsofts version av JavaScript); det tillhandahåller ramverk för skript i Windows operativsystem, vilket möjliggör automatisering av olika uppgifter och systemadministration.

Här är några viktiga punkter om Windows Script Host:

1.  **Skriptspråk:** WSH stöder flera skriptspråk, där VBScript och JScript är de vanligaste; skript kan skrivas på något av dessa språk för att automatisera uppgifter, interagera med operativsystemet eller manipulera data.
    
2.  **Skriptkörning:** WSH-skript lagras vanligtvis i filer med filtillägg som .vbs för VBScript eller .js för JScript; när du kör ett skript tolkar och kör WSH koden.
    
3.  **Automation:** WSH används ofta för automatiseringsändamål som att automatisera repetitiva uppgifter, hantera systeminställningar och utföra administrativa funktioner; är särskilt användbart för systemadministratörer och avancerade användare.
    
4.  **Konsol- och GUI-skript:** WSH-skript kan vara antingen konsolbaserade eller GUI-baserade; konsolskript körs i ett kommandoradsfönster medan GUI-skript kan skapa grafiska användargränssnitt för mer interaktiva applikationer.
    
5.  **Security:** WSH has security features to help prevent malicious scripts from causing harm; users are typically prompted before executing scripts and scripts are subject to certain restrictions to prevent unauthorized access and actions.
    
6.  **Windows Script Host Object Model:** WSH tillhandahåller en objektmodell som tillåter skript att interagera med Windows operativsystem; detta inkluderar åtkomst till filsystem, registerinställningar, nätverksresurser och andra systemkomponenter.
    
7.  **Skriptkörning från kommandoraden:** Du kan köra WSH-skript från kommandoraden genom att använda cscript.exe eller wscript.exe körbara filer, beroende på om du vill köra skriptet i en konsol- eller GUI-miljö .

## Hur öppnar man filen WS?

WS-filer är vanliga textfiler, så om du vill öppna eller redigera dem kan du använda vilken textredigerare som helst, t.ex.

- **Anteckningsblock**
- **Anteckningsblock++**
- **Apple TextEdit**
- **Visual Studio Code**

Observera att om du vill köra skriptet i WS-filen kan du helt enkelt dubbelklicka på det eller använda kommandot `wscript` på kommandoraden.

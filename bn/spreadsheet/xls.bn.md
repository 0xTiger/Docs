## একটি XLS ফাইল কি?

Files with XLS extension represent Excel Binary File Format. Such files can be created by Microsoft Excel as well as other similar spreadsheet programs such as OpenOffice Calc or Apple Numbers. File saved by Excel is known as Workbook where each workbook can have one or more worksheets. Data is stored and displayed to users in table format in worksheet and can span numeric values, text data, formulas, external data connections, images, and charts. Applications like Microsoft Excel lets you export workbook data to several different formats including [PDF](/pdf/), [CSV](/spreadsheet/csv/), [XLSX](/spreadsheet/xlsx/), [TXT](/word-processing/txt/), [HTML](/web/html/), [XPS](/page-description-language/xps/), and several others. The XLS file format was replaced with a more open and structured format, XLSX, with the release of Microsoft Excel 2007. The latest versions still provide support for creating and reading XLS files, though XLSX is the first choice of use now.

## সংক্ষিপ্ত ইতিহাস

XLS was created by Microsoft for use with Microsoft Excel and is also known as Binary Interchange File Format (BIFF). This file type was introduced for the first time by making it part of Excel for Windows in 1987. XLS file format specifications were made public for the first in June 2008 as Revision 1. After that, the specifications were continuously updated and the latest revision available is as of August 2018 that is marked as Revision 8.0. A brief history of different versions of XLS file format is as follow:

* Version 7.0 (released with office 95):  This version of excel was the most robust and faster among all the versions and internal stream rewrites were updated to 32 bits.
* সংস্করণ 8 (অফিস 97 এর সাথে প্রকাশিত): VBA একটি প্রমিত ভাষা হিসাবে চালু করা হয়েছিল এবং এই সংস্করণে প্রথমবারের মতো প্রাকৃতিক ভাষা লেবেলগুলিকে সরিয়ে দেওয়া হয়েছিল। এটি প্রথমবারের মতো একটি পেপার ক্লিপ অফিস সহকারীও চালু করেছে।

* সংস্করণ 9 (অফিস 2000 এর সাথে প্রকাশিত): সংস্করণ 9-এ শুধুমাত্র ছোটখাটো পরিবর্তন ছিল যেখানে পেপার ক্লিপ অফিস সহকারী একই সাথে একাধিক বস্তু ধারণ করতে পারে যা আগে সম্ভব ছিল না।

* সংস্করণ 10 (অফিস এক্সপির সাথে প্রকাশিত): এই সংস্করণে কোনো লক্ষণীয় উন্নতি নেই।

* Version 11 (Released with office 2003): Major update in version 11, excel 2003 was the introduction of new tables.

## XLS ফাইল ফরম্যাট স্পেসিফিকেশন ##

মাইক্রোসফ্ট দ্বারা [MS-CFB]. Data is stored in the compound file by using storages, streams, and substreams that contain information about the content and structure of a workbook, including workbook data such as worksheet definitions. Each stream or substream contains a series of binary records. Each binary record contains zero or more structured fields that contain the workbook data. This section gives a brief overview of XLS File Structure, but for detailed file format specifications, one must consult the [XLS File Formation Specifications](https://msdn.microsoft.com/en-us/library/cc313154(v#office.12).aspx) নথিতে বর্ণিত যৌগিক ফাইলের আকারে বাইনারি স্ট্রীম হিসাবে ডেটা একটি XLS ফাইলে সাজানো হয়।

#### স্ট্রিম এবং সাবস্ট্রিম ####

একটি ওয়ার্কবুক ওয়ার্কবুক স্ট্রীম দ্বারা প্রতিনিধিত্ব করা হয়। একটি ওয়ার্কবুকের প্রতিটি ওয়ার্কশীট সাবস্ট্রিম দ্বারা প্রতিনিধিত্ব করা হয়। এছাড়াও, এটিতে চার্ট শীট সাবস্ট্রিম, ম্যাক্রো শীট সাবস্ট্রিম বা ডায়ালগ শীট সাবস্ট্রিম রয়েছে যা গ্লোবাল সাবস্ট্রিম অনুসরণ করে। প্রতিটি বাইনারি স্ট্রীম বা সাবস্ট্রিম যাতে ওয়ার্কবুক ডেটা থাকে তা অবশ্যই বাইনারি রেকর্ডের একটি সিরিজ হিসাবে লিখতে হবে।

#### রেকর্ড ####

একটি ওয়ার্কবুকের বৈশিষ্ট্য সম্পর্কে তথ্য একটি রেকর্ড হিসাবে সংরক্ষণ করা হয় যা বাইটের একটি পরিবর্তনশীল দৈর্ঘ্যের ক্রম। একটি বাইনারি রেকর্ড নিম্নলিখিত তিনটি উপাদান নিয়ে গঠিত:

**রেকর্ডের ধরন:** রেকর্ডের ধরন হল একটি দুই-বাইটের স্বাক্ষরবিহীন পূর্ণসংখ্যা যা নির্দিষ্ট করে যে কী ধরনের তথ্য রেকর্ড দ্বারা নির্দিষ্ট করা হয়েছে এবং এই রেকর্ডের জন্য নির্দিষ্ট ডেটার গঠন কীভাবে সাজানো এবং গঠন করা হয়েছে। রেকর্ড টাইপ মান অবশ্যই রেকর্ড গণনা (বিভাগ 2.3) থেকে একটি মান হতে হবে বা রেকর্ডটি অবশ্যই ভবিষ্যতের রেকর্ড আর্কিটেকচার (বিভাগ 2.1.6) ব্যবহার করতে হবে।

**রেকর্ডের আকার**: রেকর্ডের আকার হল একটি দুই-বাইটের স্বাক্ষরবিহীন পূর্ণসংখ্যা যা বাইটের গণনা নির্দিষ্ট করে যা রেকর্ড ডেটার মোট আকার নির্দিষ্ট করে। রেকর্ডের আকার অবশ্যই 0 এর থেকে বেশি বা সমান এবং 8224 এর থেকে কম বা সমান হতে হবে।

**রেকর্ড ডেটা:** রেকর্ড ডেটা কম্পোনেন্টে এমন ক্ষেত্র থাকে যা একটি নির্দিষ্ট রেকর্ডের প্রকারের সাথে মিলে যায় এবং রেকর্ডের অবশিষ্ট অংশ থাকে। একটি প্রদত্ত রেকর্ড প্রকারের ক্ষেত্রের ক্রম এবং কাঠামো সেই রেকর্ডের প্রকারের জন্য সংশ্লিষ্ট বিভাগে নির্দিষ্ট করা হয়েছে। রেকর্ড ডেটা উপাদানের আকার অবশ্যই রেকর্ড আকারের সমান হতে হবে। রেকর্ড ডেটা উপাদানের ক্ষেত্রগুলিতে সাধারণ মান, মানগুলির অ্যারে, বিভিন্ন ক্ষেত্রের কাঠামো, ক্ষেত্রগুলির অ্যারে এবং কাঠামোর অ্যারে থাকতে পারে।

#### সেল টেবিল ####

কোষ হল একটি ওয়ার্কবুকের মৌলিক ব্লক যা ওয়ার্কবুকের বিষয়বস্তু যেমন পাঠ্য, সূত্র এবং সংখ্যাসূচক ডেটা সংরক্ষণ করে। কোষগুলি সেল টেবিল নামে একটি ডেটা কাঠামোর মাধ্যমে সঞ্চিত ডেটার রেকর্ড বজায় রাখে। সেল টেবিল নিজেই রেকর্ডের ক্রমানুসারে সংরক্ষণ করা হয় যা স্পেসিফিকেশন নথিতে সংজ্ঞায়িত CELLTABLE নিয়মগুলি মেনে চলে। এটি সারি ব্লকগুলির একটি সিরিজ নিয়ে গঠিত যেখানে সারিগুলি সারি ব্লকগুলিতে সাজানো হয়। প্রতিটি সারি ব্লকে প্রথম সারি থেকে ডেটা ধারণকারী শেষ সারি পর্যন্ত ডেটা থাকে।

ডেটা বা সারি বিন্যাস প্রতিটি সারি ব্লকের জন্য একটি সারি রেকর্ডে সংরক্ষণ করা হয়। ডেটা বা পৃথক সেল বিন্যাস ধারণ করা প্রতিটি কক্ষ একটি রেকর্ড দ্বারা প্রতিনিধিত্ব করা হয়। একটি কক্ষের সাথে সম্পর্কিত বিন্যাস পৃথক সেল বিন্যাস, সারি বিন্যাস, কলাম বিন্যাস, বা ডিফল্ট সেল বিন্যাস থেকে উদ্ভূত হতে পারে। বিন্যাসের জন্য অগ্রাধিকারের ক্রম হল সর্বোচ্চ অগ্রাধিকার সহ পৃথক সেল বিন্যাস, তারপর সারি বিন্যাস, এবং তারপর কলাম বিন্যাস, এবং তারপর ডিফল্ট সেল বিন্যাস। যে কক্ষগুলিতে ডেটা নেই এবং পৃথক বিন্যাস নেই সেগুলি সংরক্ষণ করা হয় না৷

#### সূত্র ####

একটি সূত্র হল একটি কক্ষের মান, সেল রেফারেন্স, নাম, ফাংশন বা অপারেটরগুলির একটি ক্রম যা একসাথে একটি নতুন মান তৈরি করে। সূত্রগুলি একটি টোকেনাইজড উপস্থাপনায় সংরক্ষণ করা হয় যা পার্সড এক্সপ্রেশন নামে পরিচিত। একটি পার্সড এক্সপ্রেশন প্রদর্শন এবং ব্যবহারকারী সম্পাদনার জন্য রানটাইমে একটি পাঠ্য সূত্রে রূপান্তরিত হয়। কোষ সূত্র সূত্র রেকর্ড দ্বারা নির্দিষ্ট করা হয়. অ্যারে সূত্রগুলি অ্যারে রেকর্ড দ্বারা নির্দিষ্ট করা হয়। ভাগ করা সূত্রগুলি ShrFmla রেকর্ড দ্বারা নির্দিষ্ট করা হয়।

#### চার্ট ####

চার্ট শীট একটি চার্ট নির্দিষ্ট করে, একটি গ্রাফিক যা ডেটা প্রদর্শন করে বা ভিজ্যুয়াল আকারে ডেটার সেটগুলির মধ্যে সম্পর্ক এবং একটি চার্ট ডেটা ক্যাশে, চার্ট ডেটাতে ব্যবহৃত ডেটার একটি স্থানীয় অনুলিপি অনুপস্থিত বা বহিরাগত লিঙ্কগুলির সাথে তথ্য উৎস ভাঙ্গা হয়. চার্টটি এক বা দুই-অক্ষের গোষ্ঠী নির্দিষ্ট করে, চার্টের ডেটার বিরুদ্ধে প্লট করা অক্ষের একটি সেট এবং তালিকায় উল্লেখ করা সিরিজ, ট্রেন্ডলাইন এবং ত্রুটি বারের সেট। প্রতিটি অক্ষ গোষ্ঠী এক থেকে চারটি চার্ট গোষ্ঠী নির্দিষ্ট করে যা ডেটা প্রদর্শনের জন্য ব্যবহৃত ভিজ্যুয়ালাইজেশনের ধরন নির্দিষ্ট করে। প্রতিটি সিরিজ, ট্রেন্ডলাইন এবং ত্রুটি বার একটি চার্ট গ্রুপ নির্দিষ্ট করে যার সাথে এটি যুক্ত।

#### মেটাডেটা ####

মেটাডেটা হল একটি নির্দিষ্ট সেল বা এর বিষয়বস্তুর সাথে যুক্ত অতিরিক্ত ডেটা। মেটাডেটা শুধুমাত্র ভবিষ্যতে সম্প্রসারণযোগ্য উদ্দেশ্যে BIFF8 এ রেকর্ড করা হয়।

#### পিভট টেবিল ####

একটি PivotTable হল সেই ডেটার বিতরণের একটি ওভারভিউ পেতে উৎস ডেটার সংক্ষিপ্তকরণের একটি প্রক্রিয়া। একটি PivotTable-এ, উৎস ডেটার প্রযোজ্য কলামগুলি ক্ষেত্র হয়ে যায় যা ডেটা সংক্ষিপ্ত করতে ব্যবহার করা যেতে পারে। যখন PivotTable-এর সোর্স ডেটা OLAP সোর্স ডেটা হয়, তখন OLAP শ্রেণীবিন্যাস এবং কিছু অন্যান্য OLAP সত্তা পিভটটেবলের ক্ষেত্রে পরিণত হয়।
একটি PivotTable দুটি প্রধান অংশ আছে, একটি PivotCache এবং একটি PivotTable ভিউ। একটি একক নন-OLAP PivotCache-এর উপর ভিত্তি করে একাধিক PivotTable ভিউ থাকতে পারে।

* ফন্টের বৈশিষ্ট্য (বোল্ড, ইটালিক, ফন্টের রঙ, ফন্টের আকার, ইত্যাদি...)

* বৈশিষ্ট্যগুলি পূরণ করুন (আগ্রভূমির রঙ, পটভূমির রঙ, প্যাটার্ন, গ্রেডিয়েন্ট, ইত্যাদি...)

* প্রান্তিককরণ বৈশিষ্ট্য (বাম, কেন্দ্র, ডান প্রান্তিককরণ, ইত্যাদি...)

* সীমানা বৈশিষ্ট্য (বাম, ডান, উপরে, নীচে, পুরু বা পাতলা, রঙ, ইত্যাদি...)

* সংখ্যা বিন্যাসের বৈশিষ্ট্য (তারিখ, সময়, দশমিক স্থানের সংখ্যা, ইত্যাদি...)

* সুরক্ষা বৈশিষ্ট্য (লক, লুকানো, ইত্যাদি...)


এই বৈশিষ্ট্যগুলি, সামগ্রিকভাবে, একটি নির্দিষ্ট ঘর কীভাবে প্রদর্শিত এবং মুদ্রিত হয় তা বর্ণনা করে।

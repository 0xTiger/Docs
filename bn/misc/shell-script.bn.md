## শেল স্ক্রিপ্ট কি?

শেল স্ক্রিপ্টিং একটি প্লেইন টেক্সট ফাইলে কমান্ডের একটি সিরিজ লেখা জড়িত, প্রায়ই একটি **শেল স্ক্রিপ্ট** হিসাবে উল্লেখ করা হয়। এই স্ক্রিপ্টগুলি একটি শেল দ্বারা নির্বাহ করা হয়, যা একটি কমান্ড-লাইন ইন্টারপ্রেটার। সবচেয়ে সাধারণ শাঁস অন্তর্ভুক্ত

1. ব্যাশ (বোর্ন এগেইন শেল)
2. Zsh (Z শেল)
3. মাছ।

শেল স্ক্রিপ্টগুলি সাধারণ ওয়ান-লাইনার থেকে জটিল প্রোগ্রাম পর্যন্ত হতে পারে এবং এগুলি বিভিন্ন ধরনের কাজ সম্পাদন করতে ব্যবহৃত হয়, যেমন ফাইল ম্যানিপুলেশন, সিস্টেম প্রশাসন এবং পুনরাবৃত্তিমূলক কাজগুলির স্বয়ংক্রিয়তা।

## শেল স্ক্রিপ্টিংয়ের সুবিধা:

1.  **অটোমেশন:** শেল স্ক্রিপ্টগুলি ব্যবহারকারীদের পুনরাবৃত্তিমূলক কাজগুলিকে স্বয়ংক্রিয় করতে দেয়, সময় বাঁচায় এবং মানুষের ত্রুটির সম্ভাবনা হ্রাস করে।
    
2.  **কাস্টমাইজেশন:** ব্যবহারকারীরা তাদের নির্দিষ্ট চাহিদা অনুযায়ী স্ক্রিপ্ট তৈরি করতে পারে, উচ্চ মাত্রার কাস্টমাইজেশন প্রদান করে।
    
3.  **ব্যাচ প্রসেসিং:** শেল স্ক্রিপ্টগুলি ব্যাচ প্রসেসিং কাজগুলি পরিচালনা করার জন্য দুর্দান্ত, যেখানে একাধিক কমান্ডগুলিকে ক্রমানুসারে কার্যকর করতে হবে।
    
4.  **সিস্টেম অ্যাডমিনিস্ট্রেশন:** শেল স্ক্রিপ্টগুলি সাধারণত ব্যাকআপ, লগ রোটেশন এবং সফ্টওয়্যার ইনস্টলেশনের মতো সিস্টেম অ্যাডমিনিস্ট্রেশনের জন্য ব্যবহৃত হয়।

## একটি সাধারণ শেল স্ক্রিপ্ট লেখা:

আসুন একটি মৌলিক শেল স্ক্রিপ্ট তৈরি করি যা একটি শুভেচ্ছা বার্তা প্রিন্ট করে। একটি পাঠ্য সম্পাদক খুলুন এবং `greeting.sh` নামে একটি ফাইল তৈরি করুন। নিম্নলিখিত লাইন যোগ করুন:

```
#!/bin/bash
# This is a simple shell script

echo "Hello, welcome to the world of shell scripting!"
```

ফাইলটি সংরক্ষণ করুন এবং টার্মিনালে নিম্নলিখিত কমান্ডটি চালিয়ে এটিকে কার্যকর করুন:

```
chmod +x greeting.sh
```

এখন, আপনি স্ক্রিপ্টটি চালাতে পারেন:

```
./greeting.sh
```

আউটপুট হওয়া উচিত:

```
Hello, welcome to the world of shell scripting!
```

## উবুন্টু এবং লিনাক্সে শেল স্ক্রিপ্ট চালানো:

এখন, আমরা আলোচনা করব **কীভাবে উবুন্টু এবং লিনাক্সে একটি .sh ফাইল চালাতে হয়**।

1.  **স্ক্রিপ্টটিকে এক্সিকিউটেবল করুন:** শেল স্ক্রিপ্ট চালানোর আগে, এটি এক্সিকিউটেবল কিনা তা নিশ্চিত করুন। পূর্বে দেখানো হিসাবে `chmod` কমান্ড ব্যবহার করুন।
    
2.  **স্ক্রিপ্ট ডিরেক্টরিতে নেভিগেট করুন:** একটি টার্মিনাল খুলুন এবং আপনার শেল স্ক্রিপ্ট ধারণকারী ডিরেক্টরিতে নেভিগেট করতে `cd` কমান্ড ব্যবহার করুন।
    
3.  **স্ক্রিপ্ট চালান:** টার্মিনালে `./scriptname.sh` লিখে স্ক্রিপ্টটি চালান, আপনার স্ক্রিপ্টের প্রকৃত নাম দিয়ে স্ক্রিপ্টনাম প্রতিস্থাপন করুন।
    
```
cd path/to/script
./greeting.sh
``` 

4.  **ব্যাশ কমান্ড ব্যবহার করা:** যদি আপনার স্ক্রিপ্ট `#!/bin/bash` দিয়ে শুরু হয় (একটি **শেবাং** নামে পরিচিত), তাহলে আপনি `bash` কমান্ড ব্যবহার করেও এটি চালাতে পারেন।

```
bash greeting.sh
```

## শেল স্ক্রিপ্টে $@ মানে কি?

শেল স্ক্রিপ্টে, `$@` স্ক্রিপ্টে পাস করা সমস্ত কমান্ড-লাইন আর্গুমেন্ট উপস্থাপন করে। এটি প্রায়শই পৃথক সত্তা হিসাবে আর্গুমেন্টের তালিকা উল্লেখ করতে ব্যবহৃত হয়। দ্বৈত উদ্ধৃতিগুলির মধ্যে ব্যবহার করা হলে, যেমন `$@`, এটি পৃথক আর্গুমেন্ট সংরক্ষণ করে, স্পেস এবং বিশেষ অক্ষরগুলির জন্য অ্যাকাউন্টিং।

এখানে সংক্ষিপ্ত ব্যাখ্যা:

- `$@`: স্ক্রিপ্ট বা ফাংশনে পাস করা সমস্ত অবস্থানগত প্যারামিটার (আর্গুমেন্ট) প্রতিনিধিত্ব করে। প্রতিটি যুক্তি পৃথক শব্দ হিসাবে গণ্য করা হয়.
    
- `$@`: যখন ডবল-উদ্ধৃত করা হয়, আর্গুমেন্টের বিভাজন রক্ষা করে, স্বতন্ত্র আর্গুমেন্টের মধ্যে স্পেস বা বিশেষ অক্ষরের জন্য অনুমতি দেয়।
    

এখানে ব্যাখ্যা করার জন্য সহজ উদাহরণ:

```
#!/bin/bash

# Save this script as example.sh

echo "The total number of arguments is: $#"
echo "The arguments are: $@"
echo "The arguments with double quotes are: \"$@\""
```

আপনি যখন আর্গুমেন্ট সহ এই স্ক্রিপ্টটি চালান, উদাহরণস্বরূপ:

```
bash example.sh arg1 "argument 2" arg3
```

এটা আউটপুট হবে:

```
The total number of arguments is: 3
The arguments are: arg1 argument 2 arg3
The arguments with double quotes are: "arg1" "argument 2" "arg3"
```

আপনি দেখতে পাচ্ছেন, `$@` সমস্ত আর্গুমেন্টের প্রতিনিধিত্ব করে, এবং `$@` পৃথক আর্গুমেন্ট সংরক্ষণ করে, এমনকি যদি সেগুলিতে স্পেস থাকে।

## Co je soubor MDA?

Soubor MDA je soubor zásuvného modulu používaný systémem správy relačních databází Microsoft Access (RDBMS). Používá se pro přidávání dalších funkcí k databázovým funkcím a dotazům podporovaným softwarem. Soubor MDA obsahuje zdrojový kód jazyka Visual Basic v jazyce VBA, který se spustí při volání ze softwaru. Také by vás mohl zajímat [soubor MDB](/cs/database/mdb/), což je soubor databáze Microsoft Access, který je systémem správy relačních databází (RDBMS).

## Jak otevřít soubor MDA?

Soubor MDA můžete otevřít pomocí **Microsoft Access 365**.

## O aplikaci Microsoft Access

Microsoft Access je systém pro správu relačních databází (RDBMS), který je součástí sady aplikací Microsoft Office. Poskytuje uživatelsky přívětivé rozhraní a nástroje pro vytváření a správu databází, takže je přístupný uživatelům bez rozsáhlých zkušeností s programováním nebo správou databází. Zde jsou některé klíčové body o Microsoft Access:

1. **Struktura databáze:**
    - Access používá model relační databáze, který uživatelům umožňuje organizovat data do tabulek.
    - Tabulky mohou být vzájemně propojeny pomocí vztahů, které definují, jak jsou různé tabulky propojeny.

2. **Objekty v přístupu:**
    - **Tabulky:** Ukládejte data do řádků a sloupců.
    - **Dotazy:** Získávejte data a manipulujte s nimi na základě konkrétních kritérií.
    - **Formuláře:** Vytvářejte uživatelská rozhraní pro zadávání a prohlížení dat.
    - **Sestavy:** Vytvářejte tisknutelné sestavy a souhrny na základě databázových dat.
    - **Makra:** Automatizujte úlohy v Accessu.
    - **Moduly:** Umožňují pokročilejší automatizaci a programování pomocí jazyka Visual Basic for Applications (VBA).

3. **Zadávání a ověřování dat:**
    - Formuláře zjednodušují zadávání dat tím, že poskytují uživatelsky přívětivé rozhraní.
    - Access podporuje ověřování dat pro zajištění integrity dat.

4. **Dotazování na data:**
    - Uživatelé mohou vytvářet dotazy SQL (Structured Query Language) nebo používat grafické rozhraní návrhu dotazů k extrahování konkrétních informací z databáze.

5. **Automatizace pomocí maker a VBA:**
    - Makra poskytují způsob, jak automatizovat opakující se úkoly v Accessu bez nutnosti znalosti programování.
    - Pokročilí uživatelé mohou používat VBA k vytvoření sofistikovanější automatizace, vlastních funkcí a postupů.

6. **Integrace s dalšími aplikacemi Microsoft Office:**
    - Access se dobře integruje s dalšími aplikacemi Microsoft Office, jako jsou Excel a Outlook.

7. **Zabezpečení:**
    - Access poskytuje zabezpečení na uživatelské úrovni a umožňuje správcům řídit, kdo má přístup ke konkrétním částem databáze.

8. **Možnosti nasazení:**
    - Databáze aplikace Access mohou být uloženy na místním počítači, ve sdíleném síťovém umístění nebo v cloudu (pomocí SharePointu).

9. **Omezení:**
    - Přestože je Access výkonný nástroj pro malé a středně velké databáze, nemusí být vhodný pro rozsáhlé podnikové aplikace.

10. **Verze:**
     - Microsoft Access prošel několika verzemi, přičemž nejnovější je součástí sady Microsoft 365.

Microsoft Access běžně používají jednotlivci a malé podniky k vytváření a správě databází pro úkoly, jako je správa zásob, sledování projektů a správa kontaktů. Stojí za zmínku, že pro větší a složitější databáze se organizace mohou rozhodnout pro jiné systémy správy databází na podnikové úrovni.

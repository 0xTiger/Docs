## Kas ir WS fails?

**.WS failā** operētājsistēmā Windows parasti ir izpildāms skripts, kas paredzēts izpildei, izmantojot **Windows Script Host**. Šis skripts var ietvert dažādus elementus un rutīnas, piemēram, JScript un VBScript rutīnas, kā arī XML elementus. Veicot dubultklikšķi uz .ws faila, Windows skriptu resursdators sāks failā iegultā skripta izpildi.

## Par Windows resursdatora skriptu

Windows Script Host (WSH) ir Microsoft izstrādāts skriptu resursdators, kas lietotājiem ļauj palaist skriptus, kas rakstīti tādās skriptu valodās kā VBScript (Visual Basic Scripting Edition) un JScript (Microsoft JavaScript versija); tas nodrošina ietvaru skriptēšanai Windows operētājsistēmā, ļaujot automatizēt dažādus uzdevumus un sistēmas administrēšanu.

Šeit ir daži galvenie punkti par Windows Script Host:

1.  **Skriptu valodas:** WSH atbalsta vairākas skriptu valodas, no kurām visbiežāk tiek izmantotas VBScript un JScript; skriptus var rakstīt jebkurā no šīm valodām, lai automatizētu uzdevumus, mijiedarbotos ar operētājsistēmu vai manipulētu ar datiem.
    
2.  **Skripta izpilde:** WSH skripti parasti tiek glabāti failos ar paplašinājumiem, piemēram, .vbs VBScript vai .js JScript; kad palaižat skriptu, WSH interpretē un izpilda kodu.
    
3.  **Automatizācija:** WSH bieži izmanto automatizācijas nolūkos, piemēram, atkārtotu uzdevumu automatizēšanai, sistēmas iestatījumu pārvaldīšanai un administratīvo funkciju veikšanai; ir īpaši noderīga sistēmu administratoriem un pieredzējušiem lietotājiem.
    
4.  **Konsole un GUI skripti:** WSH skripti var būt balstīti uz konsoli vai GUI; konsoles skripti tiek palaisti komandrindas logā, savukārt GUI skripti var izveidot grafiskas lietotāja saskarnes interaktīvākām lietojumprogrammām.
    
5.  **Security:** WSH has security features to help prevent malicious scripts from causing harm; users are typically prompted before executing scripts and scripts are subject to certain restrictions to prevent unauthorized access and actions.
    
6.  **Windows Script Host Object Model:** WSH nodrošina objekta modeli, kas ļauj skriptiem mijiedarboties ar Windows operētājsistēmu; tas ietver piekļuvi failu sistēmām, reģistra iestatījumiem, tīkla resursiem un citiem sistēmas komponentiem.
    
7.  **Skripta izpilde no komandrindas:** varat izpildīt WSH skriptus no komandrindas, izmantojot izpildāmos failus cscript.exe vai wscript.exe atkarībā no tā, vai skriptu vēlaties palaist konsoles vai GUI vidē. .

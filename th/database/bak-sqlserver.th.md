## ไฟล์ BAK คืออะไร??

ไฟล์ ".bak" ในบริบทของ Microsoft SQL Server เป็นรูปแบบไฟล์สำรองที่ใช้จัดเก็บสำเนาของฐานข้อมูล SQL Server ไฟล์เหล่านี้ประกอบด้วยสแน็ปช็อตของฐานข้อมูล ณ เวลาใดเวลาหนึ่ง รวมถึงสคีมา ข้อมูล และข้อมูลอื่นๆ ที่เกี่ยวข้อง สร้างขึ้นโดยใช้ฟังก์ชันการสำรองและกู้คืนข้อมูลในตัวของ SQL Server และให้บริการตามวัตถุประสงค์ที่สำคัญหลายประการ:

1. **การกู้คืนข้อมูล:** ไฟล์ .bak เป็นวิธีการกู้คืนฐานข้อมูลในกรณีที่ข้อมูลสูญหาย เสียหาย หรือมีปัญหาอื่นๆ ด้วยการคืนค่าฐานข้อมูลจากไฟล์ .bak คุณสามารถเปลี่ยนกลับเป็นสถานะก่อนหน้า ลดการหยุดทำงานและการสูญหายของข้อมูล

2. **การย้ายและการโคลน:** ไฟล์สำรองมักใช้เพื่อย้ายฐานข้อมูลระหว่างเซิร์ฟเวอร์หรือสร้างสำเนาของฐานข้อมูลเพื่อการทดสอบ การพัฒนา หรือการรายงาน พวกเขาเสนอวิธีที่สอดคล้องและมีประสิทธิภาพในการย้ายฐานข้อมูลระหว่างสภาพแวดล้อม

3. **การกู้คืนช่วงเวลา:** SQL Server อนุญาตให้คุณทำการกู้คืนช่วงเวลาโดยใช้ไฟล์ .bak ซึ่งหมายความว่าคุณสามารถกู้คืนฐานข้อมูลในช่วงเวลาที่กำหนดได้ ซึ่งอาจมีความสำคัญอย่างยิ่งต่อการปฏิบัติตามกฎระเบียบหรือการตรวจสอบข้อมูล

4. **การกู้คืนความเสียหาย:** ไฟล์ .bak เป็นส่วนสำคัญของการวางแผนการกู้คืนความเสียหาย พวกเขารับประกันว่าข้อมูลของคุณปลอดภัยและสามารถกู้คืนได้อย่างรวดเร็วในกรณีที่ฮาร์ดแวร์ขัดข้อง ภัยพิบัติทางธรรมชาติ หรือเหตุการณ์ภัยพิบัติอื่นๆ

## สร้างไฟล์ .BAK ใน SQL Server

หากต้องการสร้างไฟล์ .bak ใน SQL Server โดยทั่วไปคุณจะใช้คำสั่ง SQL Server Management Studio (SSMS) หรือ Transact-SQL (T-SQL) เช่น BACKUP DATABASE หรือ BACKUP LOG ต่อไปนี้คือตัวอย่างง่ายๆ ของวิธีที่คุณสามารถสร้างการสำรองฐานข้อมูลโดยใช้ T-SQL:

```
BACKUP DATABASE YourDatabaseName
TO DISK = 'C:\Path\To\Your\BackupFile.bak'
```

## กู้คืนไฟล์ .BAK ใน SQL Server

หากต้องการกู้คืนฐานข้อมูลจากไฟล์ .bak คุณสามารถใช้คำสั่ง RESTORE DATABASE ได้:

```
RESTORE DATABASE YourRestoredDatabaseName
FROM DISK = 'C:\Path\To\Your\BackupFile.bak'
```

## วิธีการเปิดไฟล์ BAK ใน SQL Server?

หากต้องการเปิดและเข้าถึงข้อมูลที่จัดเก็บไว้ในไฟล์ ".bak" โดยทั่วไปคุณจะต้องกู้คืนข้อมูลดังกล่าวไปยังอินสแตนซ์ Microsoft SQL Server ต่อไปนี้เป็นขั้นตอนทั่วไปในการเปิดไฟล์ ".bak" โดยใช้ SQL Server Management Studio (SSMS):

1. **เปิดตัว SQL Server Management Studio**: เปิด SSMS บนคอมพิวเตอร์ของคุณ โดยปกติคุณจะพบได้ในเมนู Start หรือค้นหา "SQL Server Management Studio"

2. **เชื่อมต่อกับอินสแตนซ์ SQL Server**: ใน SSMS ให้เชื่อมต่อกับอินสแตนซ์ SQL Server ที่คุณต้องการกู้คืนฐานข้อมูล คุณจะต้องมีสิทธิ์ที่จำเป็นในการดำเนินการนี้

3. **กู้คืนฐานข้อมูล**:

ก. ในบานหน้าต่าง Object Explorer ทางด้านซ้าย ให้ขยายอินสแตนซ์ SQL Server

ข. ขยายโหนด "ฐานข้อมูล"

ค. คลิกขวาที่ "ฐานข้อมูล" และเลือก "กู้คืนฐานข้อมูล"

4. **ระบุต้นทางและปลายทาง**:

ก. ในหน้า "ทั่วไป" ของกล่องโต้ตอบ "กู้คืนฐานข้อมูล" ให้ป้อนชื่อฐานข้อมูลใหม่ในช่อง "ถึงฐานข้อมูล" นี่จะเป็นชื่อของฐานข้อมูลที่กู้คืน

ข. ในส่วน "แหล่งที่มา" เลือก "อุปกรณ์" เป็นประเภทสื่อสำรองข้อมูล

ค. คลิกปุ่ม "..." ข้างช่อง "Device" เพื่อค้นหาไฟล์ ".bak" ที่คุณต้องการกู้คืน

ง. เลือกไฟล์ ".bak" ที่คุณต้องการเปิดแล้วคลิก "ตกลง"

5. **ตัวเลือกการคืนค่า**: ตรวจสอบและกำหนดค่าตัวเลือกการคืนค่าตามความจำเป็น คุณสามารถระบุว่าจะเขียนทับฐานข้อมูลที่มีอยู่ ตั้งค่าตัวเลือกการกู้คืน และอื่นๆ ตรวจสอบให้แน่ใจว่าได้ตั้งค่าตัวเลือกเหล่านี้ตามความต้องการของคุณ

6. **เริ่มต้นการคืนค่า**: เมื่อคุณกำหนดค่าตัวเลือกการคืนค่าแล้ว ให้คลิกปุ่ม "ตกลง" ในกล่องโต้ตอบ "กู้คืนฐานข้อมูล" SQL Server จะเริ่มกระบวนการคืนค่า

7. **เข้าถึงฐานข้อมูลที่กู้คืน**: หลังจากกู้คืนสำเร็จ คุณจะสามารถเข้าถึงฐานข้อมูลที่กู้คืนใน SQL Server Management Studio ได้เหมือนกับฐานข้อมูลอื่นๆ คุณสามารถเรียกใช้แบบสอบถาม ดูตาราง และทำงานกับข้อมูลภายในฐานข้อมูลได้

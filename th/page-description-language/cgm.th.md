## ไฟล์ CGM คืออะไร?? ##

Computer Graphics Metafile (CGM) เป็นรูปแบบ metafile มาตรฐานสากลที่ไม่ขึ้นกับแพลตฟอร์มสำหรับจัดเก็บและแลกเปลี่ยนกราฟิกแบบเวกเตอร์ (2D) กราฟิกแรสเตอร์ และข้อความ CGM ใช้วิธีการเชิงวัตถุและข้อกำหนดฟังก์ชันมากมายสำหรับการผลิตภาพ CGM ใช้ลักษณะเฉพาะเชิงวัตถุเหล่านี้ในการปรับปรุงองค์ประกอบกราฟิกเพื่อแสดงภาพ metafile มีข้อมูลที่จำเป็นซึ่งกำหนดไฟล์อื่นๆ ใน CGM ไฟล์ต้นฉบับแบบข้อความประกอบด้วยองค์ประกอบกราฟิกทั้งหมดที่สามารถรวบรวมเป็นไฟล์ไบนารีได้ในภายหลัง โดยพื้นฐานแล้ว CGM เป็นวิธีอำนวยความสะดวกในการแลกเปลี่ยนข้อมูลแบบกราฟิก 2 มิติ โดยไม่ขึ้นกับแพลตฟอร์มหรืออุปกรณ์ใดโดยเฉพาะ

รูปแบบ CGM จัดเตรียมองค์ประกอบต่างๆ เพื่อทำหน้าที่ต่างๆ และกำหนดวัตถุเพื่อปรับค่าดั้งเดิมทางเรขาคณิตและข้อมูลกราฟิก แม้ว่า CGM จะถูกแทนที่ด้วยรูปแบบอื่นๆ ในการแสดงภาพกราฟิกบนหน้าเว็บ เนื่องจากหน้าเว็บไม่รองรับเป็นอย่างดี แต่ก็ยังเป็นที่นิยมอย่างมากในแอปพลิเคชันอุตสาหกรรม การบิน และด้านเทคนิคอื่นๆ แม้ว่า World Wide Web Consortium ได้พัฒนา WebCGM ซึ่งเป็นแนวทางทางเลือกสำหรับการใช้ CGM บนเว็บ การใช้ CGM หลักเป็นภาพประกอบของลำดับการทำงานพื้นฐานของ Graphical Kernel System (GKS) ไม่ถูกนำมาใช้ในการออกแบบระดับมืออาชีพมากนัก แต่ถูกแทนที่ด้วยรูปแบบอื่นเช่น DXF และ SVG

## ประวัติศาสตร์ ##

CGM กลายเป็นมาตรฐานสากลในปี 1987 (ISO 8632-1987) และยังได้รับการรับรองเป็นมาตรฐานแห่งชาติในสหราชอาณาจักรโดย BSI และสหรัฐอเมริกาโดย ANSI หลังจากแก้ไขหลายครั้งในปี 1991 มาตรฐาน CGM ฉบับแก้ไขก็ได้รับการเผยแพร่ในปี 1992 (ISO 8632:1992) ในปี 2544 World Wide Web Consortium ได้พัฒนา WebCGM ด้วยความสามารถที่เพิ่มขึ้นเพื่อใช้กับเว็บเพจ ในปี 2550 WebCGM รุ่นที่สองเปิดตัวและรุ่นที่สามเปิดตัวในปี 2553 พร้อมความสามารถที่เพิ่มขึ้น

## รูปแบบไฟล์ CGM ##

เมตาไฟล์กราฟิกของคอมพิวเตอร์เป็นฐานข้อมูลโดยทั่วไปสำหรับข้อมูลกราฟิกและเป็นสื่อกลางในการจับภาพ จัดเก็บ และส่งข้อมูลกราฟิก ดังนั้นจึงต้องมีส่วนประกอบของระบบกราฟิกสำหรับสร้างฐานข้อมูลพร้อมกันไปพร้อมกับการดำเนินการของแอปพลิเคชันในรูปแบบเมตาไฟล์ ในกรณีส่วนใหญ่ ส่วนประกอบนี้คือตัวสร้าง Metafile นอกจากนี้ ยังมีความต้องการส่วนประกอบอื่นที่สามารถดึงข้อมูล ตีความ และแสดงข้อมูลกราฟิกในเมตาไฟล์ได้ ความต้องการนี้ได้รับการเติมเต็มโดยการมีล่ามเมตาไฟล์ รูปต่อไปนี้แสดงถึงสภาพแวดล้อมการทำงานของเมตาไฟล์แบบกราฟิก

ความสัมพันธ์ของ CGM กับส่วนประกอบอื่นๆ ของระบบกราฟิกทั่วไปแสดงไว้ในภาพด้านบน นอกจากนี้ยังเห็นได้จากตัวเลขว่าการทำงานของ metafile ไม่ได้ขึ้นอยู่กับเอาต์พุตของอุปกรณ์ขั้นสุดท้าย

โดยทั่วไป metafile มีสองประเภท: **การจับภาพส่วน** และ **การจับภาพ** ฟังก์ชันหลักของเมตาไฟล์การจับภาพคือการจับภาพคำจำกัดความของรูปภาพหลายภาพโดยไม่ขึ้นกับอุปกรณ์ ในขณะที่ metafiles การจับภาพเซสชันใช้อินเทอร์เฟซระบบเพื่อจับภาพการสนทนาเอาต์พุตในระบบกราฟิก CGM อยู่ในหมวดหมู่ของเมตาไฟล์การจับภาพแบบคงที่ CGM จัดให้มีการจัดวางส่วนประกอบอย่างเป็นระเบียบด้วยโครงสร้างสองระดับ

CGM matches the needs of both communication-dependent as well as performance-dependent applications. Centralized and distributed graphics systems can use CGM in an unlimited number of ways.  It can be tailored to access graphics devices using a spooling system.

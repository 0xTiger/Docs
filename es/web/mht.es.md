## ¿Qué es un archivo MHT?

Un archivo con la extensión .mht es un formato de archivo de archivo habilitado para MIME que contiene diferentes tipos de datos en un solo archivo. Puede almacenar datos como texto, imágenes, estilo de página en forma de archivos [CSS](/es/web/css/), JavaScript y otros recursos como recursos incrustados en él. Los archivos MHT, que tienen un mensaje de tipo MIME/rfc822, encapsulan todo el contenido de un archivo HTML como un único archivo para almacenarlo en dispositivos de almacenamiento. Las aplicaciones de software como Microsoft Word le permiten convertir sus documentos WORD a MHT exportándolos como archivos MHT. Los archivos MHT se pueden abrir con navegadores populares como Microsoft Internet Explore y Google Chrome.

## Formato de archivo MHT

Al igual que [MHTML](/es/web/mhtml/), MHT también es una encapsulación MIME de documentos HTML agregados. El contenido del documento dentro de un documento MHT, como imágenes en línea, hojas de estilo, subprogramas, etc., están codificados con MIME cuando están vinculados a un recurso raíz a través de URI. Los clientes de correo electrónico como Microsoft Outlook almacenan correos electrónicos (generalmente [EML](/es/email/eml/)) en formato de archivo MHT. Las especificaciones completas del formato de archivo MHT están disponibles en [RFC 822] (https://tools.ietf.org/html/rfc822) y se pueden consultar para el desarrollo de aplicaciones de software que pueden procesar archivos MHT.

## Diferencia entre MHT y MHTML

Al guardar una página en línea, se le pide al usuario que establezca el tipo de formato de archivo en el que se debe guardar la página en línea en el sistema nativo. Por lo general, el usuario se confunde entre el lenguaje de marcado y los formatos de archivo MHTML. Les resulta molesto ver que el formato de archivo es más saludable entre estos.

Cuando un usuario decide evitar desperdiciar una página en línea como lenguaje de marcado, se forma una carpeta que contiene múltiples archivos para varios contenidos de la página, es decir, unidades de área de archivos totalmente diferentes creadas para texto, gráficos, applets, etc. Por el contrario, guardar la página en línea como MHTML crea un archivo para la página en línea completa. Todos los elementos de la página, junto con gráficos, enlaces y archivos alternativos, están integrados en un solo archivo.

Naturalmente, es sencillo manejar archivos MHT o MHTML porque el archivo puede protegerse y compartirse fácilmente a través del correo electrónico. Sin embargo, la unidad de área de archivos de lenguaje de marcado bajo la posibilidad de pérdida de información, ya que la pérdida o eliminación de incluso un archivo podría hacer que la carpeta de lenguaje de marcado completa sea inútil para el usuario.

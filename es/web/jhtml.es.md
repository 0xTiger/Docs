## ¿Qué es un archivo JHTML?

Un archivo con extensión .jhtml es un archivo [HTML](/es/web/html/) con código [Java](/es/programming/java/) que se ejecuta en el servidor cuando un cliente solicita esta página en un navegador. El servidor procesa las solicitudes, ejecuta las funciones de Java contenidas en la función y devuelve la página al navegador del cliente. Los objetos Java incrustados en las páginas JHTML se ejecutan en el servidor para gestionar las solicitudes de páginas de este tipo. Los archivos JHTML también pueden acceder a la información de la base de datos utilizando la conexión JDBC (Java [Database](/es/database/) Connectivity). Los archivos JHTML se pueden abrir en cualquier editor de texto y se pueden ver en navegadores web como Google Chrome, Firefox y Safari.

## Formato de archivo JHTML

JHTML es una tecnología patentada de ATG y se puede crear utilizando el editor de documentos de Dynamo de ATG (Art Technology Group). Los archivos JHTML se escriben en formato de archivo de texto sin formato utilizando el código HTML y Java estándar. Estos contienen etiquetas HTML estándar además de etiquetas propietarias que hacen referencia a objetos Java. Cuando el usuario solicita una página de este tipo, el servidor HTTP reenvía la solicitud a un servidor de aplicaciones Java. La página JHTML se convierte primero en un archivo .java y se compila para generar un archivo [.class](/es/programming/class/) que se ejecuta como un servlet. Esto genera un flujo de datos HTTP y HTML estándar que se devuelven al navegador solicitante para mostrarlos al usuario. Esto es útil para ejecutar consultas relacionadas con la base de datos en el servidor y devolver el resultado acumulado final al navegador del cliente.

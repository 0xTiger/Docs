## ¿Qué es un archivo IN?

Un archivo IN es un archivo de entrada que utiliza GNU Autoconf al crear una aplicación. Cuando se ejecuta el proceso de configuración, un script Autoconf (archivo .AC) puede hacer referencia a uno o más archivos ".in" o ".h.in". Los archivos IN definen variables a las que se hace referencia durante la instalación del programa, como el número de versión y la fecha de lanzamiento de una aplicación. Los archivos IN también se pueden usar para determinar la presencia o ausencia de funciones en una instalación en particular. Se pueden usar aplicaciones como Notepad y Notepad++ para abrir y editar archivos IN.

## Formato de archivo IN - Más información

Autoconf es un componente del sistema de compilación GNU que genera compilaciones personalizadas de aplicaciones para varias plataformas. Es una de varias herramientas de compilación de GNU. Otras herramientas incluyen Automake, Gnulib y Libtool. Sin necesidad de intervención manual del usuario, estos scripts pueden adaptar los paquetes a una amplia gama de sistemas tipo UNIX. Autoconf genera un script de configuración del paquete a partir de un archivo de plantilla que enumera las funciones del sistema operativo que el paquete puede usar en forma de llamadas de macro M4

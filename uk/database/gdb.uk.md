## Що таке файл GDB?

Файл ESRI База геоданих (FileGDB) — це колекція файлів у папці на диску, яка містить пов’язані геопросторові дані, такі як набори даних об’єктів, класи об’єктів і пов’язані таблиці. Щоб він працював, поряд із файлом .gdb у тому ж каталозі потрібно зберігати певні інші файли. Запити можна виконувати у файлі .gdb для керування як просторовими, так і непросторовими даними.

## Формат файлу GDB – більше інформації

Файлові бази геоданих складаються із семи системних таблиць і даних користувача. Дані користувача можуть зберігатися в таких типах наборів даних:

* Клас ознак
* Функціональний набір даних
* Набір даних мозаїки
* Каталог растрів
* Набір растрових даних
* Схематичний набір даних
* Стіл (непросторовий)
* Ящики для інструментів

Набори даних об’єктів можуть містити класи об’єктів, а також такі типи наборів даних:

* Додатки
* Анотація, пов’язана з функціями
* Геометричні мережі
* Мережні набори даних
* Посилкові тканини
* Класи стосунків
* Території
* Топології

Максимальний розмір наборів даних у файлових базах геоданих за замовчуванням становить 1 ТБ. Максимальний розмір можна збільшити до 256 ТБ для великих наборів даних (зазвичай растрових). Це керується ключовим словом конфігурації. Для отримання додаткової інформації див. Ключові слова конфігурації для файлових баз геоданих.

Файлові бази геоданих також можуть містити підтипи та домени та брати участь у реплікації перевірки/реєстрації та односторонніх репліках.

Файлова база геоданих може бути доступна одночасно декільком користувачам. Якщо користувачі редагують, вони повинні редагувати різні набори даних.

## Специфікації формату файлу GDB ##

Файл GDB є власним форматом ESRI, і його характеристики недоступні для всіх. З цієї причини деталі формату файлу для FIle GDB не можуть бути задокументовані ніде, окрім деяких джерел, які зробили це [частково](https://github.com/rouault/dump_gdbtable/wiki/FGDB-Spec) шляхом зворотного проектування .

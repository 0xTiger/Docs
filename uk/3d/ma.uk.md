## Що таке файл MA?

Файл із розширенням .ma — це файл тривимірного проекту, створений за допомогою програми Autodesk Maya. Він містить великий список текстових команд для вказівки інформації про файл. Файл **.ma** можна відкрити та відредагувати в будь-якому текстовому редакторі, щоб виправити будь-які проблеми з командами, якщо файл буде пошкоджено. Ці файли містять інформацію для визначення інформації про 3D-сцену, таку як геометрія, освітлення, анімація та візуалізація.

## Формат файлу MA

Файли MA зберігаються на диск у текстовому форматі ASCII, на відміну від файлів MB, які зберігаються на диск у двійковому форматі. Докладну довідку про формат файлу MA можна знайти в [документації Autodesk Maya](https://download.autodesk.com/us/maya/2010help/index.html?url=Glossary_M_ma_file_format.htm,topicNumber=d0e192001) для довідки розробника.

### MA Заголовок файлу

Заголовок файлу MA починається з розділу коментарів, який містить інформацію про створення файлу та дату зміни. Програми для читання файлів Maya ігнорують цей блок, оскільки він використовується лише з інформаційною метою. Однак заголовок має починатися з перших шести символів як "//Maya".

Заголовок файлу ASCII виглядає наступним чином.

```
//Maya ASCII 1.0 scene
//Name: solstice.ma
//Last modified: Sun, Dec 21, 97 10:18:26 AM
```
### Посилання на файли

Розділ посилань на файли файлу .ma містить інформацію про всі інші файли Maya, на які є посилання в цьому файлі. Кожен файл, на який посилається, можна прочитати за допомогою однієї команди файлу, яка міститься в тому самому файлі. Синтаксис команди файлу, яка використовується для посилання:

```
file -r -rpr prefixString fileName;
```
або

```
file -r -ns nameSpace fileName
```
Ось приклад рядка.

```
file -r -rpr "solar" "/u/sally/work/solar.ma";
```
У цьому прикладі показано, що об’єкт сонця, який міститься у файлі `solar`, буде доступний у цьому файлі за допомогою імені "solar_sun".

### Вимоги

Розділ вимог у файлі .ma складається з серії обов’язкових команд і повідомляє May інформацію, таку як інформація про версію та плагін, яка потрібна для читання файлів. Нижче наведено приклад розділу вимог.

```
requires maya "2.0";
requires specialPlugIn "1.2";
```


Наступний розділ визначає вимоги. Він складається з ряду команд, що вимагають. Цей розділ файлу повідомляє Maya, яке програмне забезпечення потрібне для належного завантаження файлу. Зокрема, яку версію Maya та які плагіни.

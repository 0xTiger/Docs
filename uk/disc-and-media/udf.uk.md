## Що таке файл UDF?
Файл із розширенням .udf — це формат образу диска, який зазвичай використовується для збереження файлів на оптичних носіях; можна використовувати для запису DVD, CD та інших оптичних носіїв; зберігає колекцію файлів, використовуючи структуру каталогів, визначену стандартом UDF; дозволяє видаляти та змінювати файли на цільовому диску навіть після їх запису. Асоціація оптичних технологій зберігання даних встановила файлову систему UDF як стандарт для створення спільної файлової системи для всіх оптичних носіїв, наприклад для перезаписуваних оптичних носіїв і носіїв лише для читання.

## Формат файлу UDF
Формат файлу UDF — це відкрита файлова система, яка не залежить від виробника, для зберігання комп’ютерних даних для широкого діапазону носіїв. Він зазвичай використовується для DVD-дисків і нових форматів оптичних дисків. Зазвичай програмне забезпечення освоює файлову систему UDF у пакетному процесі та записує її на оптичний носій за один прохід. Однак, коли він записує пакети на перезаписуваний носій, UDF дозволяє створювати, видаляти та змінювати файли на диску, подібно до файлової системи загального призначення на знімних носіях, таких як флеш-накопичувачі або дискети.
### Специфікації UDF
Стандарт UDF визначає такі три варіанти файлової системи:

- **Звичайна збірка**: це оригінальний формат, який підтримується в усіх версіях UDF. Він був представлений у першій версії стандарту, цей формат можна використовувати на будь-якому типі диска, який забезпечує довільний доступ для читання або запису, наприклад DVD+RW, жорсткі диски та носії DVD-RAM.
- **VAT build**: використовується спеціально для запису на одноразовий носій. VAT - це додаткова структура на диску, яка дозволяє записувати пакети; тобто перевідображення фізичних блоків, коли файли або інші дані на диску змінюються або видаляються. Для носіїв з одноразовим записом весь диск віртуалізовано, що робить характер одноразового запису прозорим для користувача; з диском можна поводитись так само, як із перезаписуваним диском.
- **Spared (RW) build**: використовується спеціально для запису на перезаписуваний носій. Він додає додаткову резервну таблицю для керування несправностями, які згодом виникнуть на частинах диска, які були перезаписані кілька разів. Ця таблиця зберігає відстеження зношених секторів і переставляє їх на робочі. Управління дефектами UDF не застосовується до систем, які вже реалізують іншу форму керування дефектами, наприклад Mount Rainier (MRW) для оптичних дисків або дисковий контролер для жорсткого диска.

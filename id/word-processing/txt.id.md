## Apa itu file TXT?

File dengan ekstensi .TXT mewakili dokumen teks yang berisi teks biasa dalam bentuk garis. Paragraf dalam dokumen teks dikenali oleh carriage return dan digunakan untuk pengaturan konten file yang lebih baik. Dokumen teks standar dapat dibuka di editor teks apa pun atau aplikasi pengolah kata pada sistem operasi yang berbeda. Semua teks yang terkandung dalam file tersebut dalam format yang dapat dibaca manusia dan diwakili oleh urutan karakter.

File teks dapat menyimpan data dalam jumlah besar karena tidak ada batasan ukuran konten. Namun, editor teks yang membuka file besar seperti itu harus pintar memuat dan menampilkannya. Hampir semua sistem operasi dilengkapi dengan editor teks yang memungkinkan Anda membuat dan mengedit file teks. Misalnya, OS Windows hadir dengan Notepad dan Wordpad untuk tujuan ini. Demikian pula, MacOS hadir dengan TextEdit untuk membuat dan mengedit Dokumen Teks. Namun, ada editor teks gratis lain yang tersedia juga di internet yang memberi Anda kemampuan untuk bekerja dengan Dokumen Teks seperti Notepad ++ yang jauh lebih canggih dalam hal fungsionalitas.

## Spesifikasi Format File ##

Format file teks tidak memiliki spesifikasi format file khusus. File teks memiliki tipe MIME "teks/polos" dan memiliki sedikit atau tanpa pemformatan sama sekali. Ini memungkinkan editor teks untuk membuka file tersebut tanpa persyaratan lain. Kumpulan karakter default dari file teks adalah ASCII yang digunakan untuk membuat dan menampilkan konten file teks. Karakter dikodekan menggunakan rangkaian karakter ASCII, tetapi ini membatasi penggunaan karakter seperti Tanda Pound, Dolar, dan tanda Euro yang tidak dapat direpresentasikan menggunakan rangkaian karakter ASCII. Dengan demikian, file teks juga dapat disimpan dalam format Unicode, dengan UTF-8 yang paling banyak digunakan.

### Format Berkas Teks Windows ###

File teks pada OS Windows terdiri dari beberapa baris dimana setiap baris terdiri dari urutan karakter. Setiap baris tersirat pengguna ditentukan oleh kombinasi dua karakter yaitu carriage return (CR) dan Line Feed (LF). File teks Windows dapat dalam pengkodean ANSI, OEM, Unicode atau UTF-8. Pengkodean UTF-16 membantu menyimpan informasi dalam file teks yang memerlukan dua byte untuk representasi. File tersebut biasanya dimulai dengan Byte Order Mark (BOM) yang mengomunikasikan keberakhiran konten file. Perlu dicatat bahwa aplikasi lain di OS Windows dapat menyimpan informasi dalam format file teks tetapi dengan ekstensi file yang berbeda untuk mewakili teks khusus aplikasi. Misalnya, bahasa pemrograman biasanya menyimpan kode dalam file teks tetapi dengan ekstensinya sendiri.

### Format Berkas Teks Unix ###

Semua sistem seperti itu mendenda file teks sebagai file yang karakternya disusun menjadi nol baris atau lebih. Setiap baris adalah urutan nol atau lebih karakter non-baris baru dan karakter baris baru terminasi, biasanya LF.

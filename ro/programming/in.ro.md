## Ce este un fișier IN?

Un fișier IN este un fișier de intrare pe care GNU Autoconf îl folosește atunci când construiește o aplicație. Când se rulează procesul de configurare, un script Autoconf (fișier .AC) se poate referi la unul sau mai multe fișiere „.in” sau „.h.in”. Fișierele IN definesc variabile la care se face referire în timpul instalării programului, cum ar fi numărul versiunii unei aplicații și data lansării. Fișierele IN pot fi utilizate și pentru a determina prezența sau absența caracteristicilor într-o anumită instalație. Aplicații precum Notepad și Notepad++ pot fi folosite pentru a deschide și edita fișiere IN.

## IN Format de fișier - Mai multe informații

Autoconf este o componentă a sistemului de compilare GNU care generează versiuni personalizate de aplicații pentru diverse platforme. Este unul dintre numeroasele instrumente de construcție GNU. Alte instrumente includ Automake, Gnulib și Libtool. Fără a fi nevoie de intervenția manuală a utilizatorului, aceste scripturi pot adapta pachetele la o gamă largă de sisteme asemănătoare UNIX. Autoconf generează un script de configurare a pachetului dintr-un fișier șablon care listează caracteristicile sistemului de operare pe care pachetul le poate folosi sub formă de apeluri macro M4

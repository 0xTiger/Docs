## Kas yra WS failas?

**.WS faile** sistemoje Windows paprastai yra vykdomasis scenarijus, sukurtas vykdyti naudojant **Windows Script Host**. Šis scenarijus gali apimti įvairius elementus ir procedūras, tokias kaip JScript ir VBScript, taip pat XML elementus. Dukart spustelėjus .ws failą, Windows Script Host inicijuos į failą įterpto scenarijaus vykdymą.

## Apie „Windows Host Script“.

Windows Script Host (WSH) yra Microsoft sukurta scenarijų priegloba, leidžianti vartotojams paleisti scenarijus, parašytus tokiomis scenarijų kalbomis kaip VBScript (Visual Basic Scripting Edition) ir JScript (Microsoft JavaScript versija); ji suteikia scenarijų rašymo sistemą Windows operacinėje sistemoje, leidžiančią automatizuoti įvairias užduotis ir sistemos administravimą.

Štai keletas pagrindinių Windows Script Host punktų:

1.  **Scenarijų kalbos:** WSH palaiko kelias scenarijų kalbas, dažniausiai naudojamos VBScript ir JScript; scenarijai gali būti parašyti bet kuria iš šių kalbų, kad būtų galima automatizuoti užduotis, sąveikauti su operacine sistema arba manipuliuoti duomenimis.
    
2.  **Scenarijaus vykdymas:** WSH scenarijai paprastai saugomi failuose su plėtiniais, pvz., .vbs (VBScript) arba .js (jei JScript); kai paleidžiate scenarijų, WSH interpretuoja ir vykdo kodą.
    
3.  **Automatizavimas:** WSH dažnai naudojamas automatizavimo tikslais, pavyzdžiui, automatizuoti pasikartojančias užduotis, valdyti sistemos nustatymus ir atlikti administravimo funkcijas; yra ypač naudinga sistemos administratoriams ir patyrusiems vartotojams.
    
4.  **Konsolės ir GUI scenarijai:** WSH scenarijai gali būti pagrįsti konsolėmis arba GUI; konsolės scenarijai paleidžiami komandinės eilutės lange, o GUI scenarijai gali sukurti grafines vartotojo sąsajas interaktyvesnėms programoms.
    
5.  **Security:** WSH has security features to help prevent malicious scripts from causing harm; users are typically prompted before executing scripts and scripts are subject to certain restrictions to prevent unauthorized access and actions.
    
6.  **Windows Script Host Object Model:** WSH suteikia objekto modelį, leidžiantį scenarijus sąveikauti su Windows operacine sistema; tai apima prieigą prie failų sistemų, registro parametrų, tinklo išteklių ir kitų sistemos komponentų.
    
7.  **Scenarijaus vykdymas iš komandinės eilutės:** WSH scenarijus galite vykdyti iš komandinės eilutės naudodami vykdomuosius failus cscript.exe arba wscript.exe, atsižvelgiant į tai, ar scenarijų norite paleisti konsolėje, ar GUI aplinkoje .

## Kaip atidaryti WS failą?

WS failai yra paprasto teksto failai, todėl jei norite juos atidaryti ar redaguoti, galite naudoti bet kurį teksto rengyklę, pvz.

- **Užrašų knygelė**
-**Užrašų knygelė++**
- **Apple TextEdit**
- **Visual Studio Code**

Atkreipkite dėmesį, jei norite paleisti scenarijų WS faile, galite tiesiog dukart spustelėti jį arba naudoti komandą wscript komandinėje eilutėje.
